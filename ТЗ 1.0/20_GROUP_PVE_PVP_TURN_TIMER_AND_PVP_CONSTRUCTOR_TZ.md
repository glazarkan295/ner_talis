# ТЗ для Claude Code: групповые PVE/PVP бои, таймер хода, порядок действий союзников и PVP-конструктор

## 0. Назначение файла

Этот файл — отдельное техническое задание для Claude Code по доработке боевой системы проекта **Нер-Талис**.

Нужно добавить и настроить:

- таймер хода в групповых боях;
- правила таймера для одиночного PVE;
- порядок действий в PVE при союзниках NPC;
- порядок действий в PVE при союзниках-игроках;
- порядок действий в PVE при смешанных союзниках: NPC + игроки;
- полноценный **PVP-конструктор** в админ-панели;
- поддержку NPC-союзников в PVP на каждой стороне;
- единые настройки группового боя для PVE и PVP.

# 1. Общая логика таймера хода

## 1.1. Групповые бои

В групповых боях, где участвуют несколько игроков, на каждый ход по умолчанию даётся:

```text
100 секунд
```

Это касается:

- PVP, где участвуют игроки против игроков;
- PVP, где у сторон есть NPC-союзники;
- PVE, где в бою участвует несколько игроков;
- PVE, где игроки сражаются вместе с NPC-союзниками;
- PVE, где игроки сражаются против группы врагов и бой считается групповым.

---

## 1.2. Одиночный PVE

Если в PVE-бою участвует только один игрок, то по умолчанию таймер хода **не включается**.

Исключения возможны для:

- особых мобов;
- боссов;
- мировых боссов;
- временных событий;
- данжей;
- подлокаций;
- ловушек;
- сюжетных боёв;
- тренировочных испытаний;
- рейдовых сценариев.

Для таких случаев таймер должен настраиваться точечно в админ-панели.

---

## 1.3. Настройка таймера в админ-панели

Таймер должен настраиваться:

- глобально для всех групповых боёв;
- отдельно для PVE;
- отдельно для PVP;
- отдельно для конкретного моба;
- отдельно для конкретного события;
- отдельно для конкретной подлокации;
- отдельно для конкретного данжа;
- отдельно для конкретного мирового босса;
- отдельно для конкретного PVP-режима;
- отдельно для особого боя.

Поля настройки:

- включить таймер хода;
- время на ход в секундах;
- значение по умолчанию: 100 секунд;
- применять только в групповых боях;
- применять в одиночном PVE;
- применять только для игроков;
- применять для NPC;
- применять для всех участников;
- действие при истечении времени;
- предупреждение за N секунд до конца;
- текст предупреждения;
- текст пропуска хода;
- можно ли продлить ход;
- кто может продлить ход;
- максимальное количество продлений;
- статус настройки.

# 2. Что происходит при истечении времени

## 2.1. Возможные действия

В админ-панели нужно настроить действие при истечении таймера.

Варианты:

- пропустить ход;
- выполнить базовую защиту;
- выполнить обычный удар по случайной цели;
- выполнить заранее выбранное действие;
- повторить последнее действие;
- использовать авто-режим;
- передать ход следующему участнику;
- применить штраф;
- вывести игрока из боя, если он долго не отвечает;
- завершить бой, если все игроки неактивны.

Для PVP по умолчанию безопаснее:

```text
пропустить ход или выполнить базовую защиту
```

Для PVE можно настраивать отдельно.

---

## 2.2. Предупреждение игроку

Если включён таймер, игроку должно приходить предупреждение.

Пример:

```text
⏳ До конца хода осталось 15 секунд.
```

Настраивается:

- за сколько секунд предупреждать;
- сколько предупреждений отправлять;
- текст предупреждения;
- отправлять ли предупреждение всем или только текущему игроку;
- показывать ли таймер в сообщении боя.

# 3. Порядок действий в PVE-конструкторе

## 3.1. Новая настройка

В PVE-конструктор нужно добавить блок:

**Порядок действий в бою**

Этот блок управляет тем, кто и в каком порядке ходит, если в бою на стороне игрока есть союзники.

---

## 3.2. Сценарий 1: игрок + NPC-союзники

Если на стороне игрока есть NPC-союзники, нужно настраивать порядок:

Варианты:

- сначала игрок, потом NPC-союзники;
- сначала NPC-союзники, потом игрок;
- порядок по инициативе;
- порядок по скорости;
- порядок по роли;
- фиксированный порядок;
- случайный порядок каждый раунд;
- NPC ходят автоматически после игрока;
- NPC ходят автоматически до игрока.

Поля:

- тип порядка;
- формула инициативы;
- учитывать скорость;
- учитывать эффекты;
- учитывать оглушение;
- учитывать замедление;
- учитывать проклятия;
- показывать действия NPC игроку;
- объединять действия NPC в одно сообщение;
- отправлять действия NPC отдельными сообщениями.

---

## 3.3. Сценарий 2: несколько игроков-союзников

Если на стороне игрока есть другие игроки, бой считается групповым.

Нужно настраивать:

- порядок игроков;
- таймер хода;
- что делать, если игрок не ходит;
- может ли лидер группы выбирать цель;
- может ли каждый игрок выбирать цель сам;
- можно ли ждать всех игроков;
- можно ли ходить по очереди;
- можно ли ходить одновременно;
- как обрабатываются конфликтующие действия;
- как выводить общий лог боя.

Варианты порядка:

- строго по очереди;
- по инициативе;
- по скорости;
- все выбирают действия одновременно;
- лидер выбирает порядок;
- случайный порядок каждый раунд;
- порядок по вступлению в бой.

---

## 3.4. Сценарий 3: игроки + NPC-союзники

Если на стороне игрока одновременно есть игроки и NPC-союзники, нужно настраивать смешанный порядок.

Варианты:

- все участники сортируются по инициативе;
- сначала игроки, потом NPC;
- сначала NPC, потом игроки;
- каждый NPC привязан к конкретному игроку и ходит после него;
- NPC ходят в конце раунда;
- NPC ходят в начале раунда;
- NPC ходят автоматически между ходами игроков;
- NPC-командир задаёт отдельный порядок, если есть.

Поля:

- приоритет игроков;
- приоритет NPC;
- инициатива NPC;
- инициатива игроков;
- группировать NPC-действия;
- показывать NPC как отдельных участников;
- показывать NPC кратко;
- может ли игрок командовать NPC;
- может ли NPC выбрать цель сам;
- как NPC выбирает цель;
- что делать, если игрок-владелец NPC пропустил ход.

# 4. Порядок действий противников в PVE

В PVE-конструктор нужно добавить настройки стороны противников.

Противники могут быть:

- один моб;
- группа мобов;
- элитный моб + прислужники;
- босс + фазы;
- мировой босс;
- NPC-противники;
- смешанная группа мобов и NPC;
- событие-ловушка с действиями между раундами.

Настройки:

- порядок действий противников;
- ходят все враги или часть;
- враги ходят до игроков или после;
- враги ходят по инициативе;
- враги выбирают цель случайно;
- враги выбирают цель по агро;
- враги выбирают самого слабого;
- враги выбирают самого опасного;
- враги атакуют NPC-союзников;
- враги атакуют только игроков;
- враги могут менять цель;
- враги могут фокусить одну цель.

# 5. PVP-конструктор

## 5.1. Общая задача

В админ-панели нужно добавить полноценный раздел:

**PVP-конструктор**

PVP-конструктор должен управлять всеми настройками PVP-боя.

Логика PVP в большей степени схожа с PVE-боем, но с важными отличиями:

- игроки могут быть с обеих сторон;
- у каждой стороны могут быть NPC-союзники;
- действия игроков требуют таймера;
- нужны правила победы/поражения;
- нужны ограничения на участие;
- нужны правила наград и штрафов;
- нужны правила посмертных PVP-проклятий;
- нужна защита от злоупотреблений.

---

## 5.2. Типы PVP

Конструктор должен поддерживать типы:

- дуэль 1 на 1;
- групповая дуэль;
- команда против команды;
- свободное PVP;
- арена;
- заказ на игрока;
- охота за наградой;
- клановый бой;
- гильдейский бой;
- оборона локации;
- нападение на караван;
- бой события;
- бой в подлокации;
- бой за точку;
- будущий рейдовый PVP;
- особый сюжетный PVP.

---

## 5.3. Стороны PVP-боя

В PVP должны быть стороны.

Сторона может содержать:

- одного игрока;
- несколько игроков;
- NPC-союзников;
- призванных существ;
- временных помощников;
- мобов события;
- защитников локации;
- наёмников.

Поля стороны:

- название стороны;
- список игроков;
- список NPC-союзников;
- лимит игроков;
- лимит NPC;
- кто лидер;
- кто выбирает цели;
- порядок действий;
- условия победы;
- условия поражения;
- можно ли покинуть бой;
- что происходит при выходе;
- таймер хода;
- штрафы за пропуск;
- статус стороны.

---

## 5.4. NPC-союзники в PVP

В PVP с каждой стороны могут участвовать NPC-союзники.

Настройки NPC-союзников:

- какие NPC доступны;
- кто может взять NPC;
- лимит NPC на сторону;
- лимит NPC на игрока;
- стоимость найма;
- условия найма;
- длительность участия;
- поведение NPC;
- порядок хода NPC;
- выбор цели;
- можно ли командовать NPC;
- может ли NPC умереть;
- что происходит после смерти NPC;
- награда/штраф за смерть NPC;
- можно ли лечить NPC;
- действуют ли на NPC эффекты игроков.

---

## 5.5. Таймер в PVP

В PVP таймер хода включён по умолчанию.

Базовое значение:

```text
100 секунд на ход
```

Настраивается:

- глобально для PVP;
- отдельно для режима PVP;
- отдельно для арены;
- отдельно для события;
- отдельно для дуэли;
- отдельно для группового боя;
- отдельно для стороны;
- отдельно для особого PVP-боя.

Поля:

- время на ход;
- предупреждение за N секунд;
- действие при пропуске;
- сколько пропусков допустимо;
- штраф за пропуск;
- авто-защита;
- авто-атака;
- исключение из боя;
- техническое поражение;
- можно ли продлить ход;
- кто может продлить ход.

# 6. Общая логика хода в PVP

## 6.1. Порядок действий

PVP-конструктор должен поддерживать порядок:

- строго по очереди;
- по инициативе;
- по скорости;
- по уровню;
- случайный порядок;
- все игроки выбирают одновременно;
- сначала сторона A, потом сторона B;
- чередование сторон;
- NPC после своего игрока;
- NPC в конце раунда;
- NPC по инициативе вместе со всеми;
- особый порядок для события.

---

## 6.2. Одновременные действия

Если включён режим одновременного выбора действий:

- все игроки выбирают действие в пределах таймера;
- после окончания таймера система рассчитывает итог;
- действия сортируются по инициативе/скорости/приоритету;
- если цель умерла до действия, выбирается fallback;
- если игрок не выбрал действие, применяется правило пропуска.

---

## 6.3. Лог боя

PVP должен иметь понятный лог боя.

Настройки:

- показывать всем полный лог;
- показывать каждой стороне свой лог;
- скрывать часть действий противника;
- показывать действия NPC;
- объединять мелкие действия;
- отправлять лог одним сообщением;
- отправлять лог несколькими сообщениями;
- показывать итог раунда;
- показывать таймер следующего хода.

# 8. Настройки победы и поражения

## 8.1. PVE

Условия победы:

- все враги побеждены;
- босс побеждён;
- игроки продержались N раундов;
- выполнена цель события;
- игроки сбежали;
- защищён NPC;
- собран предмет.

Условия поражения:

- все игроки умерли;
- ключевой NPC умер;
- истекло время;
- провалено условие события;
- игроки сбежали, если это считается поражением.

---

## 8.2. PVP

Условия победы:

- все игроки противника побеждены;
- лидер противника побеждён;
- сторона набрала очки;
- сторона удержала точку;
- сторона выполнила цель;
- противник сдался;
- противник пропустил слишком много ходов;
- техническая победа.

Условия поражения:

- все игроки стороны побеждены;
- лидер стороны побеждён;
- сторона сдалась;
- сторона покинула бой;
- слишком много пропусков;
- техническое поражение;
- провал условия события.

- поражение;
- смерть NPC;
- выход игрока из боя.

Тест не должен менять реальных игроков.

---

# 13. Сообщения игрокам

Нужно настроить тексты:

- начало боя;
- чей ход;
- сколько времени осталось;
- предупреждение таймера;
- ход пропущен;
- NPC сделал действие;
- игрок сделал действие;
- итог раунда;
- победа;
- поражение;
- техническая победа;
- техническое поражение;
- NPC погиб;
- игрок покинул бой;
- бой завершён по таймеру.

Сообщения должны приходить сразу или по заданному времени через нормальную систему доставки сообщений, а не ждать следующего сообщения игрока.

---

# 14. Проверка ошибок

Конструкторы должны проверять:

- включён групповой бой, но нет таймера;
- PVP без таймера;
- сторона PVP без участников;
- NPC-союзник не имеет поведения;
- порядок действий не задан;
- одинаковая инициатива без правила tie-break;
- действие при пропуске не задано;
- награды PVP настроены небезопасно;
- PVP-проклятие не учитывает правило достижения;
- одиночный PVE случайно получил таймер без причины;
- событие требует таймер, но он отключён;
- NPC может ходить, но не имеет навыков;
- лог боя слишком длинный;
- сообщение превышает лимиты Telegram/VK.

---

# 15. Критерии готовности

Задача считается выполненной, если:

- в групповых PVE/PVP боях по умолчанию есть 100 секунд на ход;
- значение таймера можно точечно менять в админ-панели;
- одиночный PVE по умолчанию не имеет таймера;
- для особых одиночных PVE боёв можно включить таймер;
- в PVE-конструкторе есть порядок действий для NPC-союзников;
- в PVE-конструкторе есть порядок действий для союзников-игроков;
- в PVE-конструкторе есть порядок действий для смешанной стороны NPC + игроки;
- создан PVP-конструктор;
- PVP использует общую боевую логику с PVE там, где это возможно;
- в PVP с каждой стороны могут участвовать NPC-союзники;
- порядок действий в PVP настраивается;
- таймер в PVP работает;
- пропуск хода по таймеру работает;
- сообщения игрокам приходят вовремя;
- есть предпросмотр и тестовый бой;
- все поля, статусы, ошибки и подсказки отображаются на русском языке.
