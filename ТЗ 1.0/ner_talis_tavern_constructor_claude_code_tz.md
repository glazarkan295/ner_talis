# Нер-Талис — ТЗ для Claude Code
# Конструктор таверны в админ-панели

Версия: 1.0  
Назначение: создать в админ-панели полноценный конструктор таверны для Telegram/VK-бота и сайта/профиля проекта Нер-Талис.

---

## 1. Главная цель

Нужно сделать конструктор таверны, через который админ сможет создавать, редактировать, импортировать и настраивать таверны в городе, крепости, поселениях, временных лагерях, особых локациях и событиях.

Таверна должна быть не просто статичной кнопкой, а отдельной настраиваемой игровой сущностью с:

- описанием;
- кнопками;
- NPC;
- меню еды и напитков;
- отдыхом;
- слухами;
- заданиями;
- событиями;
- временными посетителями;
- репутацией;
- услугами;
- ценами;
- эффектами;
- ограничениями;
- рисками;
- текстами сообщений бота;
- связями с другими конструкторами.

---

## 2. Общие требования

Конструктор таверны должен находиться в админ-панели и быть связан с конструктором города, локаций, NPC, предметов, эффектов, заданий, репутации, событий, кнопок и текстов.

Все поля, категории, типы, статусы и действия в админ-панели должны отображаться на русском языке.

Все тексты, которые получает игрок от бота, должны редактироваться в админ-панели.

Все кнопки таверны должны быть отдельными редактируемыми сущностями.

## 4. Типы таверн

В конструкторе должны быть типы таверн:

1. **Городская таверна**  
   Обычная таверна в городе. Может давать еду, отдых, слухи, задания и NPC.

2. **Портовая таверна**  
   Таверна у пристани. Может иметь слухи о море, рыбалке, торговле, кораблях, портовом рынке и криминальных событиях.

3. **Таверна крепости**  
   Безопасное место в крепости, где игрок может отдохнуть, получить слухи, встретить наёмников, разведчиков, охотников или координаторов.

4. **Дорожная таверна**  
   Таверна между локациями. Может быть временной точкой отдыха, слухов и случайных встреч.

5. **Гильдейская таверна**  
   Место для заданий, рейдов, найма команды, слухов о мировых боссах и подземельях.

6. **Скрытая таверна**  
   Открывается по условиям: репутация, квест, предмет, скрытая метка, достижение.

7. **Событийная таверна**  
   Работает только во время мирового/сезонного события.

8. **Временная таверна**  
   Появляется на ограниченное время в лагере, караване, осаде, фестивале или рейдовом событии.


- режим рынка.

### Формула цены

```text
base_price = service.price

level_modifier = 1 + player_level_price_percent * player_level / 100
reputation_modifier = 1 - reputation_discount_percent / 100
event_modifier = 1 + event_price_modifier_percent / 100

final_price = base_price * level_modifier * reputation_modifier * event_modifier
final_price = max(min_price, ceil(final_price))
```

Формулы игроку не показывать. Игрок видит только итоговую цену.

## 22. Валидация

Конструктор должен проверять:

- есть ли у таверны локация;
- есть ли текст входа;
- есть ли кнопка возврата;
- все ли кнопки ведут к существующим действиям;
- все ли NPC существуют;
- все ли предметы меню существуют;
- все ли эффекты существуют;
- все ли задания существуют;
- все ли слухи имеют текст;
- нет ли услуги без цены, если она платная;
- нет ли цены ниже 0;
- нет ли недоступной услуги без текста ошибки;
- нет ли расписания без fallback-текста;
- нет ли события без условий и шанса;
- нет ли бесконечной выдачи награды без лимита;
- нет ли риска без текста для игрока;
- нет ли скрытой услуги без условия открытия;
- нет ли циклического перехода без кнопки выхода.

## 27. Acceptance criteria

Задача считается выполненной, если:

1. В админ-панели есть отдельный конструктор таверны.
2. Можно создать таверну и привязать её к городу/кварталу/локации.
3. Можно редактировать описание, картинку и текст входа.
4. Можно настраивать кнопки таверны.
5. Можно добавлять услуги.
6. Можно добавлять меню еды и напитков.
7. Можно настраивать отдых в таверне.
8. Можно настраивать слухи.
9. Можно привязывать NPC.
10. Можно привязывать задания.
11. Можно настраивать события таверны.
12. Можно настраивать цены и модификаторы.
13. Можно настраивать открытую и скрытую репутацию.
14. Можно задавать риски и последствия.
15. Можно задавать расписание.
16. Есть предпросмотр сообщений и кнопок.
17. Есть валидация ошибок.
18. Есть импорт старых таверн из кода.
19. Все тексты редактируются.
20. Все кнопки редактируются.
21. Игрок видит только доступные по условиям услуги.
22. Формулы не показываются игроку.
23. Все действия логируются.
24. Таверна работает в Telegram и VK.
25. Старое поведение не ломается после импорта.

## 28. Что нельзя делать

Нельзя:

- оставлять кнопки таверны хардкодом;
- оставлять тексты таверны хардкодом;
- делать таверну только как текстовую заглушку;
- терять старые ID при импорте;
- делать услуги без условий и валидации;
- показывать игроку формулы;
- смешивать скрытую репутацию с открытой без настройки видимости;
- давать бесконечные награды без лимитов;
- делать удаление без подтверждения;
- ломать Telegram/VK-совместимость;
- делать только одну таверну без возможности создавать другие.

## 29. Краткая формулировка задачи

Создать в админ-панели Нер-Талис полноценный конструктор таверны. Таверна должна быть отдельной сущностью, привязанной к городу, кварталу или локации, с редактируемыми текстами, кнопками, услугами, меню еды и напитков, отдыхом, слухами, NPC, заданиями, событиями, ценами, репутацией, скрытой репутацией, рисками, расписанием, предпросмотром, импортом из старого кода и логированием. Все действия игрока в таверне должны работать через Telegram/VK-бота, а все настройки должны изменяться через админ-панель.
