## Важные конфликты и риски

1. **Published vs draft**  
   В правках Codex отмечено, что редактирование опубликованных world/site/achievement объектов может менять live без правильного publish-flow или наоборот убирать объект из runtime. Это нужно исправить до крупных конструкторов.

2. **Импорт**  
   Есть критичный риск: неизвестные kind в import endpoint могут запускать импорт всего проекта. Также update-mode может демотировать published объекты в draft. Это вынесено в P1.

3. **Курьер и передача**  
   В ТЗ есть новая передача предметов, но Codex нашёл много рисков дублирования/потери предметов, денег и outbox-сообщений. Реализацию передачи нельзя расширять, пока не исправлены атомарность, идемпотентность, инвентарь и money/money_copper.

4. **Профиль игрока**  
   ТЗ требует конструктор раскладки профиля. Codex указывает, что runtime применяет tabs, но не применяет blocks. Нужно реализовать полное применение опубликованной раскладки.

5. **PVE награды**  
   ТЗ требует конструкторы мобов/дропа/опыта. Codex указывает, что live PVE battles игнорируют constructor mob rewards. Это нужно исправить.

6. **Сайт**  
   Публичный сайт нужен, но Codex указывает на риск cross-kind edit и public visibility. Нужно сначала защитить kind и visibility.

## Рекомендуемый порядок

1. Codex P1/P2.
2. Права, сессии, токены, outbox.
3. Import/draft/publish/versioning.
4. Общий UI shell админ-панели.
5. Общая библиотека компонентов: поля, autocomplete, подсказки, dynamic fields, preview.
6. Конструктор сайта/профиля/предметов/эффектов.
7. PVE/формулы/уровни/опыт.
8. Остальные конструкторы.
