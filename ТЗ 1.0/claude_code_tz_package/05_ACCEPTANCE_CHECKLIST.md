# 05. Acceptance Checklist — Нер-Талис

## 1. Безопасность и данные

- [ ] P1/P2 замечания Codex обработаны или явно закрыты комментарием с причиной.
- [ ] Курьер не дублирует предметы/монеты при сбоях уведомлений.
- [ ] Курьер не теряет предметы при переполненном инвентаре получателя.
- [ ] Курьер обновляет `money` и `money_copper` согласованно.
- [ ] Нельзя отправлять bound/protected/quest/locked предметы.
- [ ] Outbox не очищается до успешной отправки.
- [ ] SQLite outbox не теряется при конкурентных update.
- [ ] Read-only админ не получает edit-token профиля.
- [ ] Админ edit-token не сохраняется как обычная player session.

## 7. Медиа-загрузки и изображения

- [ ] Загрузчик принимает PNG, JPG/JPEG и WebP.
- [ ] GIF, SVG, BMP/TIFF, поддельные расширения и повреждённые изображения запрещены.
- [ ] Проверка выполняется через Pillow по сигнатуре и содержимому файла.
- [ ] Максимальный размер файла 8 МБ соблюдается.
- [ ] Минимальное разрешение 1×1 проверяется.
- [ ] Максимальная сторона 4096 px проверяется.
- [ ] Максимальная площадь 16 777 216 px проверяется.
- [ ] PNG нормализуется в RGBA optimized PNG.
- [ ] JPEG нормализуется в RGB quality 90.
- [ ] WebP нормализуется в RGBA quality 90.
- [ ] Метаданные удаляются при перекодировании.
- [ ] Изображения сообщений Telegram/VK принимают только локальные `/assets/...`.
- [ ] Внешние URL в конструкторе сообщений запрещены.
- [ ] Админские загрузки сохраняются в `data/public_uploads/assets/admin_uploads/<category>/<key>.<ext>`.
- [ ] Имена файлов нормализуются: латиница, нижний регистр, `_`, без спецсимволов и выхода за каталог.
- [ ] В загрузчике есть предпросмотр, размер файла, разрешение, итоговый формат и понятные ошибки.
- [ ] Создаются оптимизированные варианты 256×256 и 1024×1024.
- [ ] Есть рекомендации/подсказки по размерам для иконок, моделей, фонов, новостей, сообщений, NPC и баннеров.
- [ ] Тяжёлые изображения можно оптимизировать для телефонов.
