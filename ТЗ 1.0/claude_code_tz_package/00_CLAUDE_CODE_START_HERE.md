# 00. Claude Code — начать здесь

## Главная задача

Подготовить и доработать проект **Нер-Талис** по загруженным ТЗ.  
Сначала исправить критичные баги, затем расширять админ-панель и конструкторы.

## Рабочий порядок

### Этап 0 — обязательно перед новыми функциями

Исправить критичные замечания Codex:

- курьер и передача предметов не должны дублировать/терять предметы и деньги;
- outbox/очередь сообщений не должна терять сообщения;
- админ-токены и права не должны давать read-only пользователю редактировать профиль;
- импорт не должен случайно импортировать всё при ошибочном списке kind;
- update/import не должны переводить опубликованные объекты в draft и убирать их из игры;
- сайт не должен отдавать hidden/authorized материалы публично;
- constructor mobs/rewards должны использовать live-данные конструктора в PVE.

Подробный список: `01_CODEX_CRITICAL_FIXES.md`.

- права доступа и роли.

## Общие правила реализации

- Все пользовательские и админские названия — на русском языке.
- Технические ID можно хранить в коде/БД, но в интерфейсе показывать русские названия.
- Игроку не показывать формулы.
- Изображения, видео и файлы загружать как файлы проекта, а не как внешние ссылки.
- Для изображений выполнить требования `07_MEDIA_UPLOAD_REQUIREMENTS.md`: PNG/JPG/WebP, проверка Pillow по содержимому, лимиты 8 МБ/4096 px/16 777 216 px, нормализация и производные размеры.
- Опасные действия требуют подтверждения.
- Все импорты должны сохраняться после выхода и повторного входа.
- Ручные правки после импорта нельзя затирать без подтверждения.
- Published-объекты не должны исчезать из runtime при обычном редактировании.
