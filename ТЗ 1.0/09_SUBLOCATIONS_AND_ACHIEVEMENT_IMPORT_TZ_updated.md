# ТЗ для Claude Code: конструктор подлокаций и импорт достижений

## 1. Общая задача

Нужно добавить в админ-панель проекта **Нер-Талис** новый раздел:

**Конструктор подлокаций**

Подлокация — это ограниченное по размеру место внутри основной локации, города, крепости или события, куда игрок может попасть через переход, кнопку, событие, предмет, NPC или условие.

Примеры подлокаций:

- пещера;
- подземелье;
- лабиринт;
- дом;
- руины;
- данж;
- старая шахта;
- башня;
- погреб;
- катакомбы;
- заброшенное здание;
- скрытая комната;
- лагерь;
- временная зона мирового события.

Подлокация должна иметь собственную внутреннюю структуру: входы, комнаты, коридоры, развилки, тупики, переходы, скрытые проходы, события, ресурсы, мобов, ловушки, награды и тексты.

Дополнительно нужно выполнить:

- импорт старых достижений;
- перенос эффектов старых достижений в базу эффектов;
- связь достижений с конструктором эффектов.

---

# 2. Конструктор подлокаций

## 2.1. Назначение

Конструктор подлокаций нужен, чтобы администратор мог создавать небольшие внутренние места внутри уже существующих локаций.

Главная локация может быть большой областью, например:

- Обыкновенный лес;
- Холмистые луга;
- Малое плато;
- город;
- крепость;
- мировая зона.

А подлокация — это конкретное место внутри неё:

- пещера в лесу;
- руины на Малом плато;
- подвал в доме;
- скрытый проход в крепости;
- заброшенная шахта;
- временный данж мирового события.

---

## 2.2. Основная логика

Игрок находится в основной локации и может попасть в подлокацию через:

- кнопку;
- событие;
- скрытое событие;
- предмет-доступ;
- NPC;
- задание;
- достижение;
- мировой ивент;
- случайный переход;
- ручное действие администратора.

После входа в подлокацию игрок перемещается по её внутренним узлам.

Узел — это отдельная часть подлокации:

- вход;
- коридор;
- комната;
- развилка;
- зал;
- тупик;
- лестница;
- проход;
- тайник;
- выход;
- скрытый проход;
- опасная зона;
- финальная комната;
- комната босса.

Каждый узел настраивается отдельно.

---

# 3. Пример структуры подлокации

Примерная схема пещеры:

```text
[Вход] —— [Коридор 1] —— [Развилка]
                                /        \
                       [Коридор 2]      [Коридор 3]
                            |                |
                       [Комната 1]           |
                            |                |
                       [Коридор 4]           ▼
                                      [Комната 2]
                                             |
                                             ▼
                                      [Комната 3]
```

Каждый элемент схемы должен быть отдельным узлом, который можно редактировать.

# 7. Переходы внутри подлокации

## 7.1. Переходы между узлами

В каждом узле нужно настраивать переходы.

Переход имеет:

- откуда ведёт;
- куда ведёт;
- текст кнопки;
- иконку кнопки;
- описание перехода;
- условие доступности;
- требуемый предмет;
- требуемый уровень;
- требуемое достижение;
- требуемое задание;
- требуемый эффект или отсутствие эффекта;
- шанс срабатывания;
- скрыт до выполнения условия;
- текст при успешном переходе;
- текст при запрете;
- расход энергии, если нужен;
- риск события при переходе;
- статус.

---

## 7.2. Скрытые проходы

Подлокации должны поддерживать скрытые проходы.

Скрытый проход может открываться:

- после события;
- после боя;
- после нахождения предмета;
- после разговора с NPC;
- после выполнения задания;
- при наличии достижения;
- при наличии эффекта;
- при достаточном уровне;
- с шансом во время поиска.

# 16. Импорт старых достижений

## 16.1. Общая задача

Нужно импортировать старые достижения в новый конструктор достижений.

Импорт должен сохранять:

- название достижения;
- внутренний ID;
- описание;
- скрытое или открытое;
- условия получения;
- прогресс;
- ступени, если можно определить;
- награды;
- связанные эффекты;
- связанные предметы;
- связанные события;
- связанные мобы;
- связанные локации;
- статус;
- историю импорта.

---

## 16.2. Режимы импорта

Импорт достижений должен поддерживать режимы:

- создать новые;
- обновить импортированные;
- создать копии;
- не перезаписывать изменённые вручную;
- перезаписать с подтверждением.

После импорта должен быть отчёт:

- сколько создано;
- сколько обновлено;
- сколько пропущено;
- сколько требует проверки;
- какие достижения имеют ошибки;
- какие достижения имеют неизвестные условия;
- какие достижения имеют эффекты, которых ещё нет в базе эффектов.

---

# 17. Добавление эффектов достижений в базу эффектов

Если старое достижение имеет постоянный или временный бонус, этот бонус нужно перенести в базу эффектов.

Примеры:

- бонус к опыту;
- бонус к характеристике;
- бонус к добыче;
- бонус к ремеслу;
- бонус к торговле;
- сопротивление эффекту;
- открытие функции;
- открытие скрытого события;
- изменение шанса;
- пассивное свойство игрока.

Для каждого такого бонуса нужно создать эффект в конструкторе эффектов.

---

## 17.1. Правила создания эффектов из достижений

При импорте:

- если эффект уже существует, использовать существующий;
- если эффекта нет, создать новый;
- эффект должен иметь источник **achievement**;
- эффект должен иметь связь с достижением;
- эффект должен иметь описание для игрока;
- эффект должен иметь техническое описание для админа;
- эффект должен иметь режим действия;
- эффект должен иметь условия применения;
- эффект должен иметь статус.

---

## 17.2. Связь достижения и эффекта

В карточке достижения должен быть блок:

**Эффекты достижения**

Он показывает:

- какие эффекты выдаёт достижение;
- постоянные они или временные;
- когда применяются;
- можно ли отключить;
- показывать ли игроку;
- где ещё используется эффект.

В карточке эффекта должен быть блок:

**Источник**

Он показывает:

- достижение, которое создаёт эффект;
- условия получения;
- где эффект применяется.

---

# 19. Глубина поиска в конструкторе локаций

## 19.1. Общая задача

В конструктор локаций нужно добавить механику:

**Глубина поиска**

Глубина поиска показывает, насколько далеко игрок продвинулся внутрь текущей локации за одну непрерывную серию действий **Поиск**.

Чем больше глубина поиска, тем больше нового может открываться игроку:

- новые события;
- редкие события;
- скрытые события;
- новые ресурсы;
- редкие ресурсы;
- новые мобы;
- усиленные мобы;
- скрытые переходы;
- подлокации;
- тайники;
- особые находки;
- сюжетные фрагменты;
- опасные зоны;
- финальные события локации.

---

## 19.2. Как считается глубина поиска

Глубина считается по количеству непрерывно выполненных команд:

**Поиск**

на одной и той же локации.

Пример:

- игрок нажал **Поиск** 1 раз — глубина 1;
- нажал **Поиск** 2 раза подряд — глубина 2;
- нажал **Поиск** 10 раз подряд — глубина 10.

Счётчик глубины хранится отдельно для текущей локации и текущего игрока.

---

## 19.3. Что не сбрасывает глубину поиска

Глубина поиска не должна сбрасываться, если игрок остаётся в рамках текущей локационной ветки.

Не сбрасывают глубину:

- вход в лагерь на этой локации;
- выход из лагеря обратно на эту же локацию;
- открытие профиля игрока;
- возврат из профиля;
- вход в подлокацию внутри текущей локации;
- выход из подлокации обратно в текущую локацию;
- запуск события на текущей локации;
- завершение события и возврат на текущую локацию;
- бой, который начался как событие этой локации;
- получение ресурса на этой локации;
- срабатывание ловушки этой локации;
- открытие NPC или диалога, если он относится к текущей локации.

---

## 19.4. Что сбрасывает глубину поиска

Глубина поиска сбрасывается, если игрок покинул текущую локационную ветку.

Сбрасывают глубину:

- переход на другую основную локацию;
- возврат к списку локаций;
- переход в город;
- переход в крепость;
- переход в другую глобальную область;
- ручной выход с локации;
- телепорт из локации;
- смерть и перенос в лагерь/город/другую точку, если это покидает текущую локацию;
- любой полноценный выход из текущей локации.

Правило:

**Любой выход из текущей локационной ветки сбрасывает счётчик глубины поиска.**

---

## 19.5. Настройки глубины в карточке локации

В конструкторе локаций у каждой локации должен быть блок:

**Глубина поиска**

Поля:

- использовать глубину поиска: да / нет;
- стартовая глубина;
- максимальная глубина;
- сбрасывать при выходе с локации;
- сохранять при лагере;
- сохранять при профиле;
- сохранять при подлокации;
- сохранять при событии;
- сохранять при бою внутри события;
- показывать глубину игроку: да / нет;
- текст глубины для игрока;
- техническое описание для админа;
- статус механики.

---

## 19.6. Пороги глубины

В локации можно настроить пороги глубины.

Пример:

- глубина 1–3 — обычные находки;
- глубина 4–7 — больше редких ресурсов;
- глубина 8–12 — шанс скрытых событий;
- глубина 13–20 — шанс подлокации;
- глубина 21+ — особые события, сильные мобы, редкие награды.

У каждого порога:

- название порога;
- минимальная глубина;
- максимальная глубина;
- описание;
- список доступных событий;
- список доступных ресурсов;
- список доступных мобов;
- список доступных подлокаций;
- бонус к шансам;
- штраф к шансам;
- текст для игрока;
- статус.

---

## 19.7. Связь глубины с событиями

В конструкторе событий нужно добавить условия:

- доступно с глубины;
- доступно до глубины;
- доступно в диапазоне глубины;
- повышать шанс после глубины;
- снижать шанс после глубины;
- открыть событие только при достижении глубины;
- событие срабатывает один раз на глубине;
- событие может сбросить глубину, если это указано отдельно.

Пример:

Событие **Скрытая тропа** появляется только с глубины 8.

Событие **Вы вышли к старой пещере** появляется с глубины 12 и может открыть подлокацию.

---

## 19.8. Связь глубины с ресурсами

В ресурсах локации нужно добавить условия:

- ресурс доступен с глубины;
- ресурс доступен до глубины;
- ресурс доступен в диапазоне глубины;
- шанс ресурса зависит от глубины;
- количество ресурса зависит от глубины;
- редкий ресурс открывается только на большой глубине.

Пример:

- обычные ягоды доступны с глубины 1;
- редкая трава доступна с глубины 6;
- древний осколок доступен с глубины 15.

---

## 19.9. Связь глубины с мобами

В настройках мобов локации нужно добавить:

- моб доступен с глубины;
- моб доступен до глубины;
- моб доступен в диапазоне глубины;
- шанс встречи зависит от глубины;
- ранг моба зависит от глубины;
- усиленная версия появляется после глубины;
- босс или редкий моб появляется только после нужной глубины.

Пример:

- обычный волк — с глубины 1;
- голодный волк — с глубины 5;
- старый вожак стаи — с глубины 12.

---

## 19.10. Связь глубины с подлокациями

В конструкторе подлокаций и переходов нужно добавить условие:

**Открывается с глубины поиска**

Подлокация может открываться:

- после достижения нужной глубины;
- с шансом после нужной глубины;
- через событие, которое появляется на нужной глубине;
- через скрытую кнопку, появляющуюся после нужной глубины.

Пример:

Подлокация **Старая пещера** открывается после события на глубине 10.

---

## 19.11. Тексты для игрока

Формулы и точные технические значения игроку показывать не нужно, если это не включено администратором.

Можно показывать атмосферные тексты:

- Вы углубляетесь дальше в локацию.
- Местность становится менее знакомой.
- Следы цивилизации почти исчезли.
- Впереди виднеется что-то необычное.
- Вы забрались достаточно далеко, чтобы найти то, что редко попадается обычным путникам.

В конструкторе должны быть поля:

- текст при увеличении глубины;
- текст при достижении порога;
- текст при открытии нового события;
- текст при открытии подлокации;
- текст при сбросе глубины;
- показывать текст каждый раз или только на порогах.

---

## 19.12. Отображение в админке

В предпросмотре локации нужно показывать:

- текущую тестовую глубину;
- какие события доступны на этой глубине;
- какие ресурсы доступны;
- какие мобы доступны;
- какие подлокации могут открыться;
- что будет после следующего поиска;
- что сбросит глубину.

В тестовой песочнице администратор должен уметь:

- выставить стартовую глубину;
- нажимать тестовый **Поиск**;
- видеть рост глубины;
- видеть изменение шансов;
- проверить сброс при выходе;
- проверить сохранение глубины при лагере/профиле/подлокации/событии.

---

## 19.13. Хранение состояния игрока

Для игрока нужно хранить:

- текущая локация глубины;
- текущее значение глубины;
- время последнего поиска;
- была ли глубина сброшена;
- причина сброса;
- достигнутые пороги;
- уже открытые события глубины;
- уже открытые подлокации глубины.

Если игрок перешёл на другую основную локацию, счётчик старой локации сбрасывается или закрывается по правилу выбранного режима.

---

## 19.14. Критерии готовности глубины поиска

Механика считается готовой, если:

- в конструкторе локаций появился блок **Глубина поиска**;
- глубина увеличивается при непрерывном использовании команды **Поиск** на одной локации;
- выход с локации, переход в другую локацию или список локаций сбрасывает глубину;
- лагерь, профиль, подлокация и событие внутри текущей локации не сбрасывают глубину;
- события могут открываться по глубине;
- ресурсы могут открываться по глубине;
- мобы могут появляться по глубине;
- подлокации могут открываться по глубине;
- в админке есть предпросмотр и тестовый режим;
- все поля и значения отображаются на русском языке.

# 20. Критерии готовности

Задача считается выполненной, если:

- в админ-панели появился конструктор подлокаций;
- в конструкторе локаций появилась механика глубины поиска;
- глубина поиска влияет на события, ресурсы, мобов и открытие подлокаций;
- правила сброса и сохранения глубины работают корректно;
- подлокация может быть привязана к основной локации;
- внутри подлокации можно создавать узлы;
- у каждого узла есть описание, события, ресурсы, мобы и переходы;
- можно создать схему пещеры, подземелья, лабиринта, дома, руин или данжа;
- переходы между узлами работают;
- скрытые проходы работают по условиям;
- ресурсы, мобы, ловушки и события работают внутри узлов;
- есть визуальная схема подлокации;
- есть предпросмотр Telegram/VK;
- есть тестовое прохождение в песочнице;
- старые достижения импортируются;
- эффекты старых достижений добавляются в базу эффектов;
- достижения и эффекты связаны между собой;
- после импорта создаётся понятный отчёт;
- все поля и значения отображаются на русском языке.
