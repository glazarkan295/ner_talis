# PROJECT_MAP — карта проекта Ner-Talis (для быстрой навигации)

> Назначение: читать ЭТОТ файл первым, находить нужный модуль/данные и работать
> точечно, не сканируя весь репозиторий. **При добавлении нового модуля/фичи —
> обновлять этот файл** (раздел services / data / «Где что лежит»).
> Подробный человекочитаемый обзор: `ner_talis_game_project/docs/PROJECT_STRUCTURE.md`.

Игра: Telegram + VK боты + сайт-профиль/админка (FastAPI) + React-SPA.
Контент data-driven (JSON в `data/`). Бэкенд — Python (`ner_talis_game_project/`).

## Точки входа
- `ner_talis_game_project/main.py` — запуск ботов (run_bots: Telegram/VK), восстановление таймеров (recover_runtime_timers), запуск планировщика эффектов (_start_player_effect_scheduler_once).
- `ner_talis_game_project/web_app.py` — FastAPI-приложение сайта (профиль + админка). Защита: TrustedHost, CSP/security-заголовки, rate-limit, защита аплоадов от path-traversal, опц. FORCE_HTTPS/HSTS. **OpenAPI/Swagger (/docs,/redoc,/openapi.json) по умолчанию ВЫКЛЮЧЕНЫ в проде** (ENABLE_API_DOCS=true включает для разработки). Favicon отдаётся (/favicon.svg из dist/public, /favicon.ico → 204) — без 404 в логах.
- `ner_talis_game_project/site_api.py` — API профиля игрока (frontend_profile, эндпоинты инвентаря/навыков/характеристик/использования предметов/edit-field). **Профиль V2 (ТЗ переработки профиля)**: frontend_profile дополнительно отдаёт data-driven блоки §19 — profileSchemaVersion, status (состояние/canUseInventory/canMove/restricted/таймер/штраф), warnings (штраф/эффекты/перегруз/энергия/очки), ratingPlaces (только личные места игрока), services (transfer/promo → видимость вкладки «Сервисы»), guild (None → вкладка «Гильдии» скрыта). Аддитивно — старые ключи не тронуты. Фронт-рестрактуринг (вкладки Обзор/Снаряжение/Журнал, компоненты) — следующие этапы §23.
- `ner_talis_game_project/admin_panel_api.py` — API админ-панели V1 (роуты /api/admin/...).
- `ner_talis_game_project/admin_messages_api.py` — API **очереди сообщений V2**: /api/admin/v2/messages/* (meta/stats/list/players/{id}, send (send_direct), {id}/retry, {id}/cancel, dispatch (manage_dispatcher)). Права messages.*, мутации через admin_operation. Опирается на bot_message_queue.
- `ner_talis_game_project/admin_site_api.py` — API **Конструктора сайта V2**: /api/admin/v2/site/{kind} (news/guide/faq/banner/announcement + page/page_block/menu_item/post/rating/lore/where_is/site_theme §2) CRUD + validate/publish/hide/archive/schedule. Генерик над kind через _KIND_CONFIG; права по семьям (news.*/guides.*/faq.*/ratings.*/site.homepage_edit для страниц-блоков-баннеров/site.menu_edit для меню/site.settings_edit для оформления), publish валидирует. Опирается на site_content_registry. Павильон/раскладка профиля — отдельно.
- `ner_talis_game_project/admin_uploads_api.py` — API **загрузки изображений V2** (/api/admin/v2/uploads/image): админ грузит картинку файлом (base64), она нормализуется и сохраняется в постоянный том assets/admin_uploads/CATEGORY/KEY.EXT, возвращается локальный путь (ТЗ доп.§2). Право assets.manage, аудит (asset.upload_image). Сервис: admin_panel_service.save_uploaded_image (переиспользует нормализацию/безопасный путь V1).
- `ner_talis_game_project/admin_fines_api.py` — API **Конструктора штрафов V2** (/api/admin/v2/fines/*): авторинг ТИПОВ штрафов (CRUD + validate/publish/disable/archive/delete, права fine_def.*, publish валидирует). Опирается на fine_constructor_service. Снятие/выдача штрафов игрокам — отдельно (fine_service рантайм).
- `ner_talis_game_project/admin_promos_api.py` — API **Промокодов и рассылок V2** (ТЗ §9): /api/admin/v2/promos/* (meta/list/create/delete, права promos.view/promos.manage) и /api/admin/v2/broadcast(+/preview) (права broadcast.send). Перенос V1-инструментов в RBAC-консоль V2: переиспользует admin_panel_service (create_admin_promo/delete_admin_promo/promo_list_payload — они же пишут аудит promo.create/promo.delete) и broadcast_service (select_recipient_ids/broadcast_message + аудит broadcast.send). Награды промокода — те же id каталога (вкл. синтетические монеты/опыт/очки), что и доставка наград V1.
- `ner_talis_game_project/admin_city_api.py` — API **Конструктора города и крепости V2** (ТЗ §4–§6): /api/admin/v2/city/{kind} (city_node/city_button/city_shop_item/city_service/criminal_zone) CRUD + validate/publish/disable/archive/delete + GET /tree (дерево узлов §5). Права city.view/create/edit/publish/disable/archive/delete (publish/disable/archive/delete — опасные; content ведёт черновики, economy правит товары/сервисы, read_only смотрит). Опирается на city_constructor_service. Рантайм города/крепости (city_service навигация) — отдельная подсистема.
- `ner_talis_game_project/admin_recipes_api.py` — API **Конструктора ремесла V2** (ТЗ «импорт ремесла»): /api/admin/v2/recipes/* (CRUD + validate/publish/disable/archive/delete, права recipe.*, publish валидирует) + POST /import (рецепты из crafting_recipes.json) + /{id}/usage (где используется результат). Опирается на recipe_constructor_service. Рантайм крафта — crafting_service.
- `ner_talis_game_project/admin_import_api.py` — API **единого импорта-миграции V2** (ТЗ «импорт в админку» + full-import ТЗ): /api/admin/v2/import/run (kinds+mode → constructor_import.import_all, право world.publish, опасное, аудит import.run) + **/import/dry-run** (симуляция без записи, право world.view, §3.3/§16) + **/import/report?format=json|md** (последний сохранённый отчёт, §10) + **/import/rollback** (откат последнего импорта — удаляет созданные им записи, право world.publish, опасное, аудит import.rollback, §4.1) + **/import/images** (аудит изображений §6 — image_audit_service.audit, право world.view) + /import/check (constructor_import.check_import, право world.view) + /import/meta (kinds+режимы+supportsDryRun). Массовая публикация существующего контента в конструкторы. UI — ImportSection.jsx (единая панель: типы/режим, dry-run, импорт, проверка, отчёт, feature flags).
- `ner_talis_game_project/admin_dashboard_api.py` — API **Dashboard админ-панели V2** (ТЗ 11 §16): GET /api/admin/v2/dashboard — сводка (счётчики конструкторов, ошибки/связи/картинки, последние изменения из аудита, активные мировые события, последний импорт). Доступна любому админу (главная), метрики best-effort. Опирается на admin_dashboard_service. UI — DashboardSection (стартовая вкладка «Панель состояния»).
- `ner_talis_game_project/admin_text_api.py` — API **Конструктора текстов бота V2** (full-import ТЗ §5.18): /api/admin/v2/texts/* (CRUD+lifecycle через фабрику, права text.*, POST /import → constructor_import.import_texts) + /{id}/preview и /preview (подстановка переменных). Опирается на text_constructor_service.
- `ner_talis_game_project/admin_feature_flags_api.py` — API **feature flags V2** (full-import ТЗ §14): /api/admin/v2/feature-flags GET (право system.view) + PUT (право system.manage, опасное, аудит system.feature_flag). Постепенный перевод игры на чтение V2-данных по доменам (use_v2_*). Опирается на feature_flags_service.
- `ner_talis_game_project/public_site_api.py` — **Публичный API сайта (рантайм конструктора сайта, ТЗ §2)**: /api/public/site/* (menu/theme/pages/page/{slug}/news/guides/faq/lore/where-is/ratings/banners). Только ЧТЕНИЕ и только ОПУБЛИКОВАННЫЙ контент, без авторизации, без технических полей (_kind срезается). Опирается на site_content_registry.published()/published_menu()/published_page(). Не затрагивает игровой процесс. Публичный фронт-рендер — потребитель этого API (на вырост).
- `ner_talis_game_project/admin_profile_layout_api.py` — API **Конструктора раскладки профиля V2** (ТЗ §3): /api/admin/v2/profile-layout/{kind} (profile_tab/profile_block/profile_theme) CRUD + validate/publish/disable/archive/delete. Права profile_layout.view/edit/publish (view — read_only/content; edit — content; publish/disable/archive/delete — admin/owner, опасные). Опирается на profile_layout_service. Применение раскладки к профилю (site_api) — рантайм на вырост.
- `ner_talis_game_project/admin_skills_api.py` — API **Конструктора навыков V2** (/api/admin/v2/skills/*): авторинг ОПРЕДЕЛЕНИЙ навыков (CRUD + validate/publish/disable/archive/delete, права skill_def.*, publish валидирует) + POST /import (импорт каталога путей). Опирается на skill_constructor_service. Рантайм навыков игрока (Распорядительный камень, расход ресурса/кулдауны) — active_skill_service.
- `ner_talis_game_project/admin_effect_api.py` — API **Конструктора эффектов V2**: /api/admin/v2/effects/* (CRUD + validate/publish/disable/archive/delete, права effect.*). Опирается на effect_constructor_service.
- `ner_talis_game_project/admin_item_api.py` — API **Конструктора предметов V2**: /api/admin/v2/items/* (CRUD + validate/publish/disable/archive/delete-soft/restore, права item.*, edit опубликованного → черновик+версия, hard-delete только owner с confirm=item_id и проверкой where_used) + /{id}/usage. Опирается на item_constructor_service.
- `ner_talis_game_project/admin_achievement_api.py` — API **Достижений V2**: /api/admin/v2/achievements/* (CRUD + validate/publish/disable/archive, права achievement.*, publish валидирует) и /achievements/categories/* (manage_categories). Все мутации через admin_operation. Ручная выдача/откат игрокам — на вырост (achievement engine).
- `ner_talis_game_project/admin_community_api.py` — API **Гильдий + Мировых событий V2**: /api/admin/v2/guilds/* (CRUD + activate/freeze/disband/archive + участники, права guild.*) и /api/admin/v2/events/* (CRUD + schedule/start/stop/finish/reward/archive, права world_event.*, start/schedule валидируют событие). Все мутации через run_admin_operation.
- `ner_talis_game_project/admin_world_api.py` — API **Конструктора мира V2** (/api/admin/v2/world/...): генерик над kind (location, mob, button, transition, event, npc, quest, raid + под-объекты локации: location_zone/resource/loot/mob_spawn/weekly_limit/weekly_rotation/depletion_rule/empty_event/hidden_event/event_answer + под-объекты моба: mob_variant/skill/passive/resistance/effect/event_link/zone_link/phase — те же CRUD/lifecycle, гейт world.*) + /preview (world.view) и /test-run (world.test_run). /kinds отдаёт и справочники под-систем локации (zoneTypes/resourceCategories/lootSources/weeklyLimitTypes/rotation*/redistributionModes/eventGroups/depletionTriggers) и мобов (mobVariantTypes/mobAttackTypes/mobSkillTypes/mobSkillConditions/mobBehaviorTypes/mobResistTypes). list/get/create(world.create_draft)/edit(world.edit_draft)/status/validate(world.validate)/publish(world.publish, опасное, валидирует перед публикацией)/disable/archive — каждая мутация через run_admin_operation. Опирается на world_content_registry.
- `ner_talis_game_project/admin_panel_v2_api.py` — API админ-панели **V2** (/api/admin/v2/...): RBAC-aware, аудируемые. P0: /me (роль+права), /audit (вьювер с фильтрами, право audit.view), /roles (owner-only, через admin_operation), /sessions + /sessions/revoke (список/отзыв активных сессий, токены маскируются sha256-id; права system.view/system.manage). Подключён в web_app параллельно V1. Вход: команда `/admin_panel_v2` (build_admin_panel_v2_url) + роут GET /admin_panel_v2 (тот же Vite-бандл, тот же scope активации, что и V1).
- `ner_talis_game_project/project_paths.py` — resolve_project_path / project_path (доступ к data/ и ресурсам).

## services/ (ядро логики)
- `city_service.py` — городская навигация Селдара; **process_world_action** (единый роутер действий мира: город/рынок/крафт/внешние локации/бой/штрафы/рыбалка); central_square_buttons, кварталы; хук advance_player_time; триггер city_quarter_walk.
- `external_location_service.py` — внешние локации (Холмистые луга, Обыкновенный лес, Малое плато, крепость): поиск, события (resolve_active_event), таймеры (complete_active_timer), лагерь/готовка, крепость (handle_fortress_action), claim_active_event.
- `pve_battle_service.py` — PVE-бой: handle_battle_action, ходы, урон/крит/попадание, враги (build_enemy), лут/опыт; make_player_battle_state. **Конструкторный бой (флаг WORLD_CONSTRUCTOR_LIVE)**: create_location_battle сначала пробует create_constructor_battle (бой из опубликованных location_mob_spawn: build_constructor_enemy из карточки моба, число врагов ограничено недельным запасом §22, враги тегаются source_mob_id+base_damage); enemy_raw_damage уважает base_damage конструкторного моба; grant_battle_rewards при победе списывает запас по source_mob_id. Флаг выкл или нет спаунов → легаси-бой. Работает для легаси-id локаций (normalize_battle_location сохраняет hilly_meadows/ordinary_forest/…).
- `pve_battle_models.py` — dataclass'ы боя (PlayerBattleState/EnemyBattleState с crit_chance/crit_damage), формулы calculate_hit_chance/calculate_final_damage/apply_defense.
- `derived_stats_service.py` — производные статы из атрибутов+экипировки+эффектов (calculate_player_derived_stats), модификаторы (equipment/external/passive), crit/accuracy/dodge/hp.
- `active_skill_service.py` — активные/пассивные навыки: реестр (load_active_skill_registry → data/active_skills_registry.json, v11), урон навыка, стоимость (resource_cost_with_modifiers), кулдауны, passive_stat_modifiers, weapon_requirements.
- `crafting_service.py` — мастерские (плавильня/кузница/кожевенная/ювелирная/алхимия), рецепты (data/crafting_recipes.json), WORKSHOPS, секции; ювелирка (Бижутерия→кольца(железные/серебряные)/ожерелья/рецепты), вставка камней; алхимия (неудача → suspicious_potion).
- `market_service.py` — рынки (NPC/портовый): покупка/продажа, цены, ротация портового рынка + **атомарный claim стока** (claim_port_stock); format_price.
- `fishing_service.py` — рыбалка (таймер 60с, энергия −1), трата использования удочки (gathering_tools).
- `gathering_tools.py` — прочность инструментов (удочка/топор/кирка): 10 использований/инструмент, стак 10, spend_tool_use/tool_uses_left.
- `fine_service.py` — облавы/городские штрафы, перемещение в крепость, оплата; fine_entries_for_profile (попап штрафов). Активность штрафа = `is_fine_active` (статус НЕ в INACTIVE_FINE_STATUSES: paid/removed_by_admin/expired/cancelled) — раньше проверялось только `!= paid`, из-за чего штраф с иным терминальным/битым статусом «висел» вечно (корень бага неснимаемого штрафа). Снятие/починка: `forgive_all_fines` (единый путь снятия всех штрафов админом → статус removed_by_admin, архив в removed_fines/fine_history, чистка active_fines+alias; ограничения выводятся из active_fines → снимаются автоматически, скрытых флагов нет, ТЗ §4/§5/§8), `repair_player_fines` (реконсиляция зависших данных §6/§7). Оплата у города (voluntary/overdue) и крепости (forced) — pay_fine(place).
- `small_plateau_service.py` — Малое плато: поиск (resolve_small_plateau_search, data/small_plateau_*.json), Древнее Проклятье (roll_ancient_curse_trigger, cleanse), ожог амулета, достижения (Ищущий/«Какое проклятье?» → постоянный эффект «Носитель проклятья»), PVP-хук apply_pvp_kill_postmortem_curse (посмертное проклятье на убийцу), filter_seeker_only.
- `player_time_service.py` — догон время-эффектов (часовой ожог амулета, суточные дни проклятья); advance_player_time (на действие; ставит last_activity_at), advance_all_players_time + start_persistent_player_effect_worker (фоновый планировщик).
- `battle_stimulant_service.py` — боевой стимулятор: фазы (активная/откат), зависимость, stat_percent_modifiers, статус-карточка.
- `inventory_service.py` — инвентарь: add_inventory_item, переполнение/слоты, карманы, квалити/уровень/цена генерируемых предметов.
- `item_registry.py` — реестр предметов: load_all_item_definitions, get_item_definition_by_id/by_name, build_inventory_item (агрегирует data/items_*.json).
- `currency.py` — медь как база; format_money (краткий баланс) / format_price (торговля: «13 маг. зол. 6 зол. 400 мед. монет»). Курс: сер.=1e3, зол.=1e6, маг.зол.=1e9, древн.=5e11.
- `progression_service.py` — опыт/уровни (grant_exact_experience).
- `race_bonus_service.py` — расовые бонусы (hp_multiplier, сопротивления, регенерация).
- `registration_service.py` — создание игрока (create_player), validate_name/normalize_name, load_races, гендеры; согласие перед регистрацией (consent_message, CONSENT_BUTTON, _doc_link → env LINK_PRIVACY_POLICY / LINK_TERMS_OF_SERVICE).
- `promo_service.py` — промокоды: _normalize_code (срез слэша+upper), add/delete (удаляет ВСЕ совпавшие ключи), redeem (атомарный claim_promo_use).
- `admin_panel_service.py` — логика админ-панели: каталог (HIDDEN_CATALOG_ITEM_IDS, SYNTHETIC_REWARD_IDS — монеты/очки), доставка наград (cap по меди), промо, сессии (consume_or_read_admin_session — атомарный claim токена), просмотр игроков (admin_player_detail — last_activity дд.мм.гг; get_admin_player_view_profile отдаёт editToken — профильный веб-токен для редактирования чужого профиля админом).
- `broadcast_service.py` — админская рассылка «Общее сообщение»: выбор аудитории (пол / диапазоны уровней / все / конкретные игроки), AUDIENCE_LABELS, select_recipient_ids, broadcast_message → pending_bot_messages.

## Важные инварианты (легко сломать)
- Боты ПОСЛЕ действия пересохраняют ИСХОДНЫЙ объект игрока (handlers/city.py:~113, vk). Поэтому claim-перезагрузки (события/рыбалка) надо синхронизировать В исходный объект (`player.clear(); player.update(claimed)`), иначе изменения затрутся.
- Тесты запускать из корня репозитория (иначе ложные падения проверок иконок `web/public/...`).
- Деньги хранятся в меди (64-бит) — награды монетами ограничены по медному эквиваленту (admin_panel_service.MAX_REWARD_MONEY_COPPER).
