# PROJECT_MAP — карта проекта Ner-Talis (для быстрой навигации)

> Назначение: читать ЭТОТ файл первым, находить нужный модуль/данные и работать
> точечно, не сканируя весь репозиторий. **При добавлении нового модуля/фичи —
> обновлять этот файл** (раздел services / data / «Где что лежит»).
> Подробный человекочитаемый обзор: `ner_talis_game_project/docs/PROJECT_STRUCTURE.md`.

Игра: Telegram + VK боты + сайт-профиль/админка (FastAPI) + React-SPA.
Контент data-driven (JSON в `data/`). Бэкенд — Python (`ner_talis_game_project/`).

## Точки входа
- `ner_talis_game_project/main.py` — запуск ботов (run_bots: Telegram/VK), восстановление таймеров (recover_runtime_timers), запуск планировщика эффектов (_start_player_effect_scheduler_once).
- `ner_talis_game_project/web_app.py` — FastAPI-приложение сайта (профиль + админка).
- `ner_talis_game_project/site_api.py` — API профиля игрока (frontend_profile, эндпоинты инвентаря/навыков/характеристик/использования предметов/edit-field). **Профиль V2 (ТЗ переработки профиля)**: frontend_profile дополнительно отдаёт data-driven блоки §19 — profileSchemaVersion, status (состояние/canUseInventory/canMove/restricted/таймер/штраф), warnings (штраф/эффекты/перегруз/энергия/очки), ratingPlaces (только личные места игрока), services (transfer/promo → видимость вкладки «Сервисы»), guild (None → вкладка «Гильдии» скрыта). Аддитивно — старые ключи не тронуты. Фронт-рестрактуринг (вкладки Обзор/Снаряжение/Журнал, компоненты) — следующие этапы §23.
- `ner_talis_game_project/admin_panel_api.py` — API админ-панели V1 (роуты /api/admin/...).
- `ner_talis_game_project/admin_messages_api.py` — API **очереди сообщений V2**: /api/admin/v2/messages/* (meta/stats/list/players/{id}, send (send_direct), {id}/retry, {id}/cancel, dispatch (manage_dispatcher)). Права messages.*, мутации через admin_operation. Опирается на bot_message_queue.
- `ner_talis_game_project/admin_site_api.py` — API **Конструктора сайта V2**: /api/admin/v2/site/{kind} (news/guide/faq/banner/announcement) CRUD + validate/publish/hide/archive/schedule. Права по семьям (news.*/guides.*/faq.*/site.* для баннеров), publish валидирует. Опирается на site_content_registry. Павильон/рейтинги/раскладка профиля — на вырост.
- `ner_talis_game_project/admin_effect_api.py` — API **Конструктора эффектов V2**: /api/admin/v2/effects/* (CRUD + validate/publish/disable/archive/delete, права effect.*). Опирается на effect_constructor_service.
- `ner_talis_game_project/admin_item_api.py` — API **Конструктора предметов V2**: /api/admin/v2/items/* (CRUD + validate/publish/disable/archive/delete-soft/restore, права item.*, edit опубликованного → черновик+версия, hard-delete только owner с confirm=item_id и проверкой where_used) + /{id}/usage. Опирается на item_constructor_service.
- `ner_talis_game_project/admin_achievement_api.py` — API **Достижений V2**: /api/admin/v2/achievements/* (CRUD + validate/publish/disable/archive, права achievement.*, publish валидирует) и /achievements/categories/* (manage_categories). Все мутации через admin_operation. Ручная выдача/откат игрокам — на вырост (achievement engine).
- `ner_talis_game_project/admin_community_api.py` — API **Гильдий + Мировых событий V2**: /api/admin/v2/guilds/* (CRUD + activate/freeze/disband/archive + участники, права guild.*) и /api/admin/v2/events/* (CRUD + schedule/start/stop/finish/reward/archive, права world_event.*, start/schedule валидируют событие). Все мутации через run_admin_operation.
- `ner_talis_game_project/admin_world_api.py` — API **Конструктора мира V2** (/api/admin/v2/world/...): генерик над kind (location, mob, button, transition, event, npc, quest, raid + под-объекты локации: location_zone/resource/loot/mob_spawn/weekly_limit/weekly_rotation/depletion_rule/empty_event/hidden_event/event_answer + под-объекты моба: mob_variant/skill/passive/resistance/effect/event_link/zone_link/phase — те же CRUD/lifecycle, гейт world.*) + /preview (world.view) и /test-run (world.test_run). /kinds отдаёт и справочники под-систем локации (zoneTypes/resourceCategories/lootSources/weeklyLimitTypes/rotation*/redistributionModes/eventGroups/depletionTriggers) и мобов (mobVariantTypes/mobAttackTypes/mobSkillTypes/mobSkillConditions/mobBehaviorTypes/mobResistTypes). list/get/create(world.create_draft)/edit(world.edit_draft)/status/validate(world.validate)/publish(world.publish, опасное, валидирует перед публикацией)/disable/archive — каждая мутация через run_admin_operation. Опирается на world_content_registry.
- `ner_talis_game_project/admin_panel_v2_api.py` — API админ-панели **V2** (/api/admin/v2/...): RBAC-aware, аудируемые. P0: /me (роль+права), /audit (вьювер с фильтрами, право audit.view), /roles (owner-only, через admin_operation), /sessions + /sessions/revoke (список/отзыв активных сессий, токены маскируются sha256-id; права system.view/system.manage). P1 (центр управления игроком): /players (поиск), /players/{id} (карточка + активные штрафы), /players/{id}/logs|chat|view-token, и аудируемые мутации через run_admin_operation — /rewards (rewards.grant), /message (players.message), /unstuck (players.unstuck, обходит кулдаун), /forgive-fine (fines.manage), /reset (players.reset, опасное), DELETE (players.delete, опасное, confirm=CONFIRM_DELETE). Подключён в web_app параллельно V1. Вход: команда `/admin_panel_v2` (build_admin_panel_v2_url) + роут GET /admin_panel_v2 (тот же Vite-бандл, тот же scope активации, что и V1).
- `ner_talis_game_project/project_paths.py` — resolve_project_path / project_path (доступ к data/ и ресурсам).

## services/ (ядро логики)
- `city_service.py` — городская навигация Селдара; **process_world_action** (единый роутер действий мира: город/рынок/крафт/внешние локации/бой/штрафы/рыбалка); central_square_buttons, кварталы; хук advance_player_time; триггер city_quarter_walk.
- `external_location_service.py` — внешние локации (Холмистые луга, Обыкновенный лес, Малое плато, крепость): поиск, события (resolve_active_event), таймеры (complete_active_timer), лагерь/готовка, крепость (handle_fortress_action), claim_active_event.
- `pve_battle_service.py` — PVE-бой: handle_battle_action, ходы, урон/крит/попадание, враги (build_enemy), лут/опыт; make_player_battle_state.
- `pve_battle_models.py` — dataclass'ы боя (PlayerBattleState/EnemyBattleState с crit_chance/crit_damage), формулы calculate_hit_chance/calculate_final_damage/apply_defense.
- `derived_stats_service.py` — производные статы из атрибутов+экипировки+эффектов (calculate_player_derived_stats), модификаторы (equipment/external/passive), crit/accuracy/dodge/hp.
- `active_skill_service.py` — активные/пассивные навыки: реестр (load_active_skill_registry → data/active_skills_registry.json, v11), урон навыка, стоимость (resource_cost_with_modifiers), кулдауны, passive_stat_modifiers, weapon_requirements.
- `crafting_service.py` — мастерские (плавильня/кузница/кожевенная/ювелирная/алхимия), рецепты (data/crafting_recipes.json), WORKSHOPS, секции; ювелирка (Бижутерия→кольца(железные/серебряные)/ожерелья/рецепты), вставка камней; алхимия (неудача → suspicious_potion).
- `market_service.py` — рынки (NPC/портовый): покупка/продажа, цены, ротация портового рынка + **атомарный claim стока** (claim_port_stock); format_price.
- `fishing_service.py` — рыбалка (таймер 60с, энергия −1), трата использования удочки (gathering_tools).
- `gathering_tools.py` — прочность инструментов (удочка/топор/кирка): 10 использований/инструмент, стак 10, spend_tool_use/tool_uses_left.
- `fine_service.py` — облавы/городские штрафы, перемещение в крепость, оплата; fine_entries_for_profile (попап штрафов).
- `small_plateau_service.py` — Малое плато: поиск (resolve_small_plateau_search, data/small_plateau_*.json), Древнее Проклятье (roll_ancient_curse_trigger, cleanse), ожог амулета, достижения (Ищущий/«Какое проклятье?» → постоянный эффект «Носитель проклятья»), PVP-хук apply_pvp_kill_postmortem_curse (посмертное проклятье на убийцу), filter_seeker_only.
- `player_time_service.py` — догон время-эффектов (часовой ожог амулета, суточные дни проклятья); advance_player_time (на действие; ставит last_activity_at), advance_all_players_time + start_persistent_player_effect_worker (фоновый планировщик).
- `battle_stimulant_service.py` — боевой стимулятор: фазы (активная/откат), зависимость, stat_percent_modifiers, статус-карточка.
- `inventory_service.py` — инвентарь: add_inventory_item, переполнение/слоты, карманы, квалити/уровень/цена генерируемых предметов.
- `item_registry.py` — реестр предметов: load_all_item_definitions, get_item_definition_by_id/by_name, build_inventory_item (агрегирует data/items_*.json).
- `currency.py` — медь как база; format_money (краткий баланс) / format_price (торговля: «13 маг. зол. 6 зол. 400 мед. монет»). Курс: сер.=1e3, зол.=1e6, маг.зол.=1e9, древн.=5e11.
- `progression_service.py` — опыт/уровни (grant_exact_experience).
- `race_bonus_service.py` — расовые бонусы (hp_multiplier, сопротивления, регенерация).
- `registration_service.py` — создание игрока (create_player), validate_name/normalize_name, load_races, гендеры; согласие перед регистрацией (consent_message, CONSENT_BUTTON, _doc_link → env LINK_PRIVACY_POLICY / LINK_TERMS_OF_SERVICE).
- `promo_service.py` — промокоды: _normalize_code (срез слэша+upper), add/delete (удаляет ВСЕ совпавшие ключи), redeem (атомарный claim_promo_use).
- `admin_panel_service.py` — логика админ-панели: каталог (HIDDEN_CATALOG_ITEM_IDS, SYNTHETIC_REWARD_IDS — монеты/очки), доставка наград (cap по меди), промо, сессии (consume_or_read_admin_session — атомарный claim токена), просмотр игроков (admin_player_detail — last_activity дд.мм.гг; get_admin_player_view_profile отдаёт editToken — профильный веб-токен для редактирования чужого профиля админом).

- `admin_command_service.py` / `admin_access.py` / `admin_audit.py` — админ-команды в боте, доступ (is_configured_admin_user), аудит (write_admin_audit + V2: append_admin_audit_record / read_admin_audit_records с фильтрами).
- `admin_rbac.py` — **RBAC админ-панели V2**: роли (owner/admin/support/moderator/content/economy/read_only), права (PERM_*), ROLE_PERMISSIONS, гибрид-резолюция (override в data/admin_roles.json → ENV-bootstrap owner → read_only), has_permission/require_permission, DANGEROUS_ACTIONS. **Расширенный конструктор локаций (§60)**: семьи location_resources/loot/mobs/zones/rotation/limits/links (view/edit) для новых под-систем локации (базовый цикл локаций/кнопок/событий остаётся на world.*); content/economy правят, read_only смотрит, force-вмешательство (rotation.force_update/limits.force_restore/force_deplete) — только admin/owner и помечено опасным. **Конструктор мобов (§33)**: семья mob.* (view/create/edit/publish/disable/archive/delete_soft/restore/change_image/change_stats/change_skills/change_loot/change_locations/change_events/change_spawn_chance/change_weekly_limits/test_battle/view_balance/audit); content ведёт черновики и характеристики, economy крутит дроп/лимиты, read_only смотрит; publish/disable/archive/delete_soft/restore/change_loot — опасные.
- `admin_operation.py` — **единая изменяющая админ-операция V2** (ТЗ §22): record_admin_operation / run_admin_operation → структурная запись в аудит (admin/role/target/before/after/reason/status/dangerous). Мутации V1-панели (доставка→rewards.grant, промо create/delete, смена картинки→asset.image_change, edit-session, удаление игрока→player.delete, рассылка→broadcast.send) переведены на эти записи — единый журнал с ролью и опасным флагом. read_admin_audit_records отдаёт поле `dangerous`.
- `world_content_registry.py` — **Конструктор мира V2**: генерик-реестр data-driven контента (kind→id→envelope) с жизненным циклом draft→review→ready→published→disabled→archived, переходами статусов, валидацией (VALIDATORS по kind: location/mob/button/transition/event/npc/quest — моб: боевые поля + таблица дропа; button/transition: cross-object проверка локаций + тупик через location_has_exit; event: локация+дроп-предметы+battle_mob существуют, тип/результат из справочников; npc: функции из NPC_FUNCTIONS; quest: цель по goal_type ссылается на существующий предмет/моба/локацию/NPC; raid: локация входа+босс-моб существуют, тип из RAID_TYPES) и хранилищем JSON с блокировкой. build_preview/test_run — предпросмотр локации (кнопки/переходы/события/NPC/мобы, TG/VK) и «тестовый проход» (валидация объекта + всего подграфа локации без влияния на игру) (env WORLD_CONTENT_PATH, по умолч. data/world_content.json). Чистый слой данных; аудит/права — в роутере. **Расширенный конструктор локаций**: добавлены kind'ы под-объектов локации — location_zone (зоны+защита §43–44), location_resource (§13), location_loot (§14), location_mob_spawn (§15), location_weekly_limit (§16–26), location_weekly_rotation (§35–38), location_depletion_rule (§26–30), location_empty_event (§31–34), location_hidden_event (§10), location_event_answer (§11) + справочники (ZONE_TYPES/RESOURCE_CATEGORIES/LOOT_SOURCES/WEEKLY_LIMIT_TYPES/ROTATION_*/REDISTRIBUTION_MODES/EVENT_GROUPS/DEPLETION_TRIGGERS) и валидаторы (шанс-окно min≤base, кол-ва min≤max, ссылки на локацию/моба/предмет/эффект, обязательные условия у скрытых событий/вариантов). Рантайм недельных лимитов/ротации/истощения и API/UI — следующие этапы. **Расширенный конструктор мобов**: добавлены kind'ы под-объектов моба — mob_variant (§6), mob_skill (§9), mob_passive (§10), mob_resistance (сопротивления/слабости §11), mob_effect (§12), mob_event_link (§16), mob_zone_link (§21), mob_phase (фазы босса §26); связь моб↔локация — через тот же location_mob_spawn (без дублей). MOB_TYPES расширен (§5) + справочники (MOB_VARIANT_TYPES/MOB_ATTACK_TYPES/MOB_SKILL_TYPES/MOB_SKILL_CONDITIONS/MOB_BEHAVIOR_TYPES/MOB_RESIST_TYPES). Валидатор карточки моба обогащён боевыми параметрами (§8: attack_type/behavior/min≤max в бою). Тестовый бой/симуляция баланса (§28) и UI — следующие этапы.
- `admin_entity_store.py` — **генерик-хранилище админ-сущностей V2** (для гильдий/событий): JSON-файл с блокировкой на тип, envelope (id/data/status/версия/кто-когда), управляемые переходы статусов. Чистый слой данных.
- `guild_service.py` — **Гильдии V2**: GUILD_TYPES/GUILD_ROLES, статусы draft→active→frozen→disbanded→archive, validate(), операции участников (add/remove/set_role). EntityStore(data/guilds.json).
- `world_event_service.py` — **Мировые события V2**: EVENT_TYPES, статусы draft→scheduled→active→finished/disabled→archive, validate() (даты, лимит множителей мира MAX_WORLD_MULTIPLIER). EntityStore(data/world_events.json). Этапы/глоб.прогресс/магазины/рассылки — на вырост.
- `achievement_service.py` — **Достижения V2 (авторская часть)**: справочники (ACHIEVEMENT_TYPES/RARITIES/VISIBILITIES/CONDITION_TYPES/PROGRESS_TYPES/REWARD_TYPES/REPEAT_PERIODS), статусы draft→...→published→archive, validate() (категория существует, типы из справочников, условия/награды, скрытое не раскрывает условия, item-награды есть в каталоге, ступени, даты). Два EntityStore: achievements + achievement_categories. Связан с achievement_engine (runtime).
- `achievement_engine.py` — **Achievement engine (runtime)**: состояние в player["achievements"] (earned/progress/history); record_progress(storage, player, event_type, amount, target) — хук игровых действий (обновляет прогресс по опубликованным достижениям и выдаёт при выполнении логики any/all/n_of); grant/revoke (ручная выдача/откат); apply_rewards (через _apply_rewards_to_player: опыт/монеты/очки/предметы + титулы); уведомление игроку через outbox (§20); player_view (для профиля §21, скрытые как ???) и admin_player_progress (для админ-карточки). Авто-хуки в боевой/локационный код ещё не расставлены — record_progress готов к вызову.
- `bot_message_queue.py` — **исходящая очередь сообщений V2 (мгновенная доставка)**: envelope (id/delivery_key/game_id/platform/recipient/type/text/priority/status/attempts/timestamps/error), статусы queued→sending→sent/retry_wait/failed/blocked/cancelled, дедуп по delivery_key (§11), приоритеты (§9), бэкофф повторов (§10). enqueue/list/stats/retry/cancel/dispatch_once; sender подключается через set_sender (бот регистрирует реальную отправку TG/VK). **Pluggable backend**: по умолчанию JSON-файл (env MESSAGE_QUEUE_PATH); configure_queue(storage) переключает на БД (таблица outgoing_messages row-per-message с индексами + claim_due с приоритетом, SQLite/Postgres) — вызывается на старте web/бота. JSON-хранилище остаётся на файле.
- `site_content_registry.py` — **Конструктор сайта V2 (публикуемый контент)**: генерик-реестр kind=news/guide/faq/banner/announcement, цикл draft→review→ready→scheduled→published→hidden→archived, validate (§28: заголовок/текст, запрет HTML/скриптов, даты), справочники категорий новостей/сложности гайдов/типов баннеров. EntityStore(data/site_content.json).
- `effect_constructor_service.py` — **Конструктор эффектов/зон/проклятий V2**: универсальные типы (§9: stat_modifier/resource_regeneration/max_resource_modifier/periodic_damage/control_effect/damage_response/absorb_effect/aura_effect/summon_effect/curse_effect/zone_effect/zone_protection/item_lifecycle + параметрические крит/точность/защита), общие поля (источник/цель/active_when/стак/cleanse_tags), validate (enum + тип-специфичные требования + анти-цепочки §1.2 + балансные warning, без формул в player_text). EntityStore(data/effect_constructor.json). Применение движком — на вырост.
- `item_constructor_service.py` — **Конструктор предметов V2 (авторская часть)**: справочники (категории/типы/качества/слоты/теги/свойства/эффекты), статусы draft→…→published→disabled→archive→deleted_soft, validate (§25: стак/слот/цена/двуручность/HTML), record_version (история версий), where_used (ссылки в мобах/событиях/квестах/достижениях/рецептах). EntityStore(data/item_constructor.json). Чтение игрой — runtime на вырост.
- `message_delivery.py` — **мост мгновенной доставки**: notify_player (правило одного пути — очередь при включённом диспетчере и наличии адреса, иначе старый pending_bot_messages), resolve_recipient, register_platform_sender/_combined_sender/classify_error, start_message_dispatcher (демон-цикл). Включается ENV BOT_MESSAGE_DISPATCHER_ENABLED (по умолч. выкл — поведение не меняется). Через notify_player проведены: доставка наград (deliver_rewards_to_player), уведомления достижений (achievement_engine), личное сообщение игроку (V2 player.message). Живой sender подключают main_telegram (post_init, run_coroutine_threadsafe) и vk_registration (run → self.send).

## Важные инварианты (легко сломать)
- Боты ПОСЛЕ действия пересохраняют ИСХОДНЫЙ объект игрока (handlers/city.py:~113, vk). Поэтому claim-перезагрузки (события/рыбалка) надо синхронизировать В исходный объект (`player.clear(); player.update(claimed)`), иначе изменения затрутся.
- Тесты запускать из корня репозитория (иначе ложные падения проверок иконок `web/public/...`).
- Деньги хранятся в меди (64-бит) — награды монетами ограничены по медному эквиваленту (admin_panel_service.MAX_REWARD_MONEY_COPPER).
