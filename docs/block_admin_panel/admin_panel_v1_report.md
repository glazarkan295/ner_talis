# Блок админ-чата и админ-панели V1

Источник: `ner_talis-main-51-admin-chat-progress-rewards.zip`.

## Главное изменение логики

Админ-чат больше не используется как игровой чат. Если `TELEGRAM_ADMIN_CHAT_IDS` / `VK_ADMIN_PEER_IDS` настроены, обычные игровые команды и кнопки в этом чате/беседе глушатся и не передаются в игровой роутер. Для работы админа используется сайт-админка.

Разрешённые команды админ-чата:

- `/admin_id` — показать ID чата/беседы и пользователя для настройки env.
- `/admin_help` — справка.
- `/admin_panel` — выдать одноразовую ссылку на админ-панель.

Старые текстовые команды оставлены как аварийный режим, но основная работа перенесена в админ-панель.

## Защита админ-панели

- Ссылка `/admin_panel?token=...` создаётся только из разрешённого админ-чата и только админом из env.
- URL-токен одноразовый: при первой активации удаляется и заменяется на активный `sessionToken`.
- Новая ссылка того же админа отключает старую ссылку и старую активную админ-сессию.
- Активная сессия хранится во вкладке браузера через `sessionStorage`, не в `localStorage`.
- API админ-панели требует активный админ-токен на каждом запросе.
- Для `/api/admin` добавлен отдельный rate limit.
- Общие security headers из сайта применяются и к админ-панели.

## Вкладка «Каталог»

- Показывает все предметы из `data/items_*.json`.
- Добавлены специальные ресурсы каталога:
  - `money_copper` — медные монеты;
  - `free_skill_points` — очки навыков;
  - `free_stat_points` — очки характеристик;
  - `experience_shards` — крупицы опыта.
- Есть поиск по названию/item_id и фильтр по категории.
- Карточка предмета отдаёт полное описание, raw-данные, найденные формулы, источники и назначение.
- Предмет можно выбрать для вкладки «Доставка» или «Промокоды».
- Добавлена замена изображения предмета через base64-загрузку: обновляются JSON-описания предметов/visual assets и сохранённые копии предмета в профилях.

## Вкладка «Доставка»

- Выбранные в Каталоге предметы появляются списком.
- Поиск игрока работает по нику или единому `game_id`.
- Для каждой позиции задаётся количество.
- При отправке:
  - обычные предметы идут в инвентарь;
  - монеты идут на баланс;
  - очки навыков/характеристик идут в свободные очки;
  - крупицы опыта идут в опыт 1 к 1;
  - перед изменением профиля делается backup;
  - действие пишется в admin audit log;
  - игроку добавляется очередь сообщения `pending_bot_messages` с текстом «Дар свыше» и списком наград.

## Вкладка «Промокоды»

- Создание промокода через выбранные позиции из Каталога.
- Поля: команда промокода, количество активаций, срок жизни.
- Сроки: 1 час, 12 часов, 1 день, 7 дней, 30 дней, 365 дней, бессрочный.
- Обычные предметы при активации идут в инвентарь.
- Монеты/очки/крупицы опыта идут на баланс/свободные очки/опыт.
- В списке существующих промокодов показываются код, дата создания, срок, оставшееся время, использовано/осталось, награды.

## Вкладка «Игроки»

- Поиск по нику или единому игровому ID.
- Список игроков сортируется по нику: ник, ID, уровень.
- Карточка игрока показывает базовую информацию без сырого полного JSON.
- Доступные действия:
  - удалить игрока с `CONFIRM_DELETE` на API-уровне;
  - создать специальную ссылку read-only просмотра профиля игрока;
  - посмотреть доступные логи/очередь сообщений за последние записи.
