ОБЩЕЕ ПРАВИЛО ДЛЯ ВСЕХ ФОРМУЛ:

Все итоговые расчёты используют не только базовые характеристики персонажа, но и все активные источники бонусов.

Единый итоговый бонус любого показателя считается так:

Итоговый бонус показателя =
Постоянный бонус профиля
+ Бонусы надетых вещей
+ Бонусы камней в надетых вещах
+ Бонусы зачарований надетых вещей
+ Бонусы активных сетов
+ Бонусы выпитых зелий
+ Бонусы еды и напитков
+ Баффы и дебаффы
+ Эффекты локации
+ Эффекты событий
+ Артефакты
+ Проклятия и временные эффекты

Под надетыми вещами считаются только предметы, находящиеся в слотах экипировки. Предметы в инвентаре не дают бонусов, пока не надеты или не использованы.

Под внешними воздействиями считаются любые временные эффекты, которые действуют на персонажа сейчас: выпитые зелья, еда, напитки, баффы, проклятия, эффекты зоны, эффекты погоды, эффекты события, эффекты комнаты, эффекты аномалии и другие активные модификаторы.

Если в формуле ниже написано «Бонус HP», «Бонус маны», «Бонус точности», «Бонус шанса крита» и т.п., это всегда означает уже собранный итоговый бонус из всех перечисленных источников.

Порядок расчёта:
1. Взять базовые характеристики персонажа.
2. Добавить вложенные игроком очки.
3. Собрать бонусы от надетых вещей и внешних воздействий в отдельные поля бонусов.
4. Посчитать эффективные характеристики.
5. Посчитать итоговые ресурсы, защиту, точность, уклонение, крит, регенерацию и другие показатели.
6. Применить ограничения и округления.

1. ОЧКИ ХАРАКТЕРИСТИК ЗА УРОВЕНЬ

Свободные очки характеристик = (Уровень - 1) × 5

2. ЭФФЕКТИВНАЯ ХАРАКТЕРИСТИКА

Эффективная характеристика =
floor(1000 × ln(1 + Вложенная характеристика / 1000) + Бонус характеристики)

Вложенная характеристика = Базовая характеристика + Вложенные очки
Бонус характеристики =
Бонус характеристики от надетых вещей
+ Бонус характеристики от камней, сетов и зачарований
+ Бонус характеристики от зелий, еды, напитков, баффов, проклятий, локаций, событий и других внешних эффектов

3. МЯГКИЙ УРОВЕНЬ

Мягкий уровень = floor(10 × log2(Уровень + 1))

4. ОЧКИ ЖИЗНЕЙ

Очки жизней =
ceil(
100
+ (Эфф. Выносливость × 4.0)
+ (Эфф. Сила × 0.8)
+ (Мягкий уровень × 4)
+ Бонус HP
)

Бонус HP = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

5. ОЧКИ ДУХА

Очки духа =
ceil(
20
+ (Эфф. Выносливость × 1.2)
+ (Эфф. Сила × 1.0)
+ (Эфф. Ловкость × 0.7)
+ (Мягкий уровень × 1.2)
+ Бонус духа
)

Бонус духа = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

6. ОЧКИ МАНЫ

Очки маны =
ceil(
20
+ (Эфф. Интеллект × 1.6)
+ (Эфф. Мудрость × 1.3)
+ (Мягкий уровень × 1.2)
+ Бонус маны
)

Бонус маны = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

7. ФИЗИЧЕСКАЯ ЗАЩИТА

Физическая защита =
ceil(
(Броня × 1.5)
+ (Эфф. Выносливость × 0.9)
+ (Эфф. Сила × 0.6)
+ (Эфф. Ловкость × 0.2)
+ Бонус физической защиты
)

Бонус физической защиты = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

8. МАГИЧЕСКАЯ ЗАЩИТА

Магическая защита =
ceil(
(Магическая броня × 1.5)
+ (Эфф. Мудрость × 0.9)
+ (Эфф. Интеллект × 0.6)
+ (Эфф. Выносливость × 0.2)
+ Бонус магической защиты
)

Бонус магической защиты = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

Если отдельной магической брони нет:

Магическая защита =
ceil(
(Броня × 0.8)
+ (Эфф. Мудрость × 0.9)
+ (Эфф. Интеллект × 0.6)
+ (Эфф. Выносливость × 0.2)
+ Бонус магической защиты
)

9. СНИЖЕНИЕ УРОНА ЗАЩИТОЙ

Снижение урона =
min(
0.70,
Защита / (Защита + 300 + (Мягкий уровень × 8))
)

10. ИТОГОВЫЙ ВХОДЯЩИЙ УРОН

Итоговый урон =
max(
1,
ceil(Входящий урон × (1 - Снижение урона))
)

11. ТОЧНОСТЬ

Точность =
ceil(
(Эфф. Восприятие × 1.8)
+ (Эфф. Ловкость × 1.1)
+ (Мягкий уровень × 0.7)
+ Бонус точности
)

Бонус точности = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

12. УКЛОНЕНИЕ

Уклонение =
ceil(
(Эфф. Ловкость × 1.8)
+ (Эфф. Восприятие × 0.9)
+ (Эфф. Мудрость × 0.3)
+ (Мягкий уровень × 0.5)
+ Бонус уклонения
)

Бонус уклонения = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

13. ШАНС ПОПАДАНИЯ

Шанс попадания =
Точность / (Точность + max(1, Уклонение))

Шанс попадания =
min(
0.95,
max(
0.10,
Точность / (Точность + max(1, Уклонение))
)
)

Шанс попадания % =
ceil(Шанс попадания × 100)

14. РЕСУРСЫ АКТИВНЫХ ДЕЙСТВИЙ

Отдельный параметр концентрации в профиле не используется.
Активные навыки балансируются через:
- расход очков духа;
- расход очков маны;
- откат в ходах;
- требования навыка;
- условия применения.

Если навык относится к ветви духа, он тратит дух.
Если навык относится к ветви маны, он тратит ману.
Нейтральные базовые действия могут не требовать ресурса или иметь малую стоимость по правилам конкретного навыка.

15. ШАНС КРИТИЧЕСКОГО УДАРА

Крит-стат =
ceil(
(Эфф. Восприятие × 1.5)
+ (Эфф. Ловкость × 0.8)
+ (Эфф. Мудрость × 0.2)
+ (Мягкий уровень × 0.2)
+ Бонус шанса крита
)

Бонус шанса крита = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

Шанс крита =
Крит-стат / (Крит-стат + 5000)

Шанс крита =
min(0.49, Шанс крита)

Шанс крита % =
ceil(Шанс крита × 100)

16. КРИТИЧЕСКИЙ УРОН

)

Магический критический урон % =

17. РЕГЕНЕРАЦИЯ ЗДОРОВЬЯ

Реген HP % =
min(

0.5
+ (12 × Эфф. Выносливость / (Эфф. Выносливость + 2000))

Бонус регенерации HP % = надетые вещи + камни + зачарования + сеты + зелья + еда/напитки + баффы/дебаффы + эффекты локации + артефакты + временные эффекты

20. Единая система бонусов и совместимость

20.1. Источники бонусов
Все дополнительные бонусы из предметов, камней, сетов, зачарований, зелий, баффов, артефактов, проклятий и временных эффектов должны попадать в отдельные поля бонусов персонажа.

Примеры полей:
- bonus_hp;
- bonus_spirit;
- bonus_mana;
- bonus_physical_defense;
- bonus_magic_defense;
- bonus_accuracy;
- bonus_dodge;
- bonus_crit_chance;
- bonus_crit_damage_percent;
- bonus_hp_regen_percent;
- bonus_spirit_regen_percent;
- bonus_mana_regen_percent.

Нельзя добавлять сильные бонусы напрямую к итоговому значению после применения ограничения. Сначала добавляется бонус, потом считается формула, потом применяется ограничение.

20.2. Вложенная характеристика и эффективная характеристика
Вложенная характеристика используется для открытия характеристических навыков.
Эффективная характеристика используется для расчёта силы персонажа, ресурсов, защиты, точности, уклонения, крита и навыков.

Экипировка, камни, зачарования, зелья, баффы, артефакты и временные эффекты повышают эффективную характеристику, но не считаются вложенными очками.

20.3. Зелья развития
Если зелье временно повышает характеристику, оно не открывает новые навыки.
Если особое зелье выдаёт свободные очки характеристик, эти очки сначала попадают в свободный запас. Они начнут учитываться для открытия навыков только после ручного вложения игроком.

20.4. Глобальные ограничения
Финальные ограничения системы:
- снижение входящего урона защитой: максимум 70%;
- шанс критического удара: максимум 49%;
- регенерация HP, духа и маны: максимум 80% от максимального ресурса;
- шанс попадания: минимум 10%, максимум 95%;
- итоговый входящий урон после защиты: минимум 1;

20.5. Рекомендуемый порядок расчёта
1. Посчитать вложенные характеристики.
2. Добавить бонусы к характеристикам и получить эффективные характеристики.
3. Посчитать мягкий уровень.
4. Посчитать HP, дух, ману, энергию, защиту, точность, уклонение, крит и регенерацию.
5. Добавить эффекты навыков, предметов, камней, сетов, зачарований, зелий и проклятий через соответствующие поля бонусов.
6. Применить ограничения.
7. Рассчитать итоговый урон, лечение, восстановление ресурсов или другой результат действия.

21. ПОКАЗАТЕЛЬ ЭНЕРГИИ

Энергия — отдельный внебоевой ресурс для поисков и исследования локаций.
Она не заменяет очки духа и ману.

21.1. МАКСИМАЛЬНАЯ ЭНЕРГИЯ

Базовая максимальная энергия = 100

Макс. энергия =
max(
    1,
    100 + Бонус максимальной энергии
)

Бонус максимальной энергии может идти от:
- пассивных навыков;
- снаряжения;
- камней;
- сетовых бонусов;
- зачарований;
- временных эффектов еды и напитков;
- особых событий.

21.2. ТЕКУЩАЯ ЭНЕРГИЯ

Текущая энергия не может быть меньше 0 и больше максимальной энергии.

Текущая энергия = min(Макс. энергия, max(0, Текущая энергия))

21.3. РАСХОД ЭНЕРГИИ НА ПОИСКИ

Итоговый расход энергии =
max(
    1,
    ceil(Базовый расход локации × Множитель сложности × (1 - Бонус экономии энергии % / 100))
)

Если текущей энергии меньше, чем итоговый расход:
- поиск нельзя начать;
- или поиск запускается с сильным штрафом, если это разрешено правилами конкретной локации.

Рекомендуемые расходы:
- стартовая / безопасная локация: 5-8 энергии;
- обычная полевая локация: 8-12 энергии;
- опасная локация: 12-18 энергии;
- высокоуровневая локация: 18-25 энергии;
- аномальная зона / руины Древних: 25-40 энергии.

21.4. ВОССТАНОВЛЕНИЕ ЭНЕРГИИ

Энергия восстанавливается при употреблении блюд или напитков.

Восстановление энергии =
min(
    Макс. энергия,
    Текущая энергия + ceil(Базовое восстановление блюда или напитка × (1 + Бонус восстановления энергии % / 100))
)

Примерные значения:
- лёгкая еда: +10-15 энергии;
- обычное блюдо: +20-30 энергии;
- плотное блюдо: +35-50 энергии;
- бодрящий напиток: +15-35 энергии;
- редкий походный рацион: +50-80 энергии.

21.5. ПОЛЯ ПРОФИЛЯ

Рекомендуемые поля персонажа:
- max_energy;
- current_energy;
- bonus_max_energy;
- bonus_energy_saving_percent;
- energy_restore_bonus_percent.


36. ПРАВИЛО ДЛЯ РЕАЛИЗАЦИИ В КОДЕ

Перед расчётом профиля код должен собрать единый словарь бонусов из всех активных источников:

1. Надетые предметы из equipment.
2. Камни, зачарования и свойства этих надетых предметов.
3. Активные комплекты из active_sets.
4. Активные временные эффекты из active_effects.
5. Баффы, дебаффы, зелья, еда, напитки, эффекты локации и события.

Пример итоговых полей:
- bonus_strength;
- bonus_endurance;
- bonus_agility;
- bonus_perception;
- bonus_intelligence;
- bonus_wisdom;
- bonus_hp;
- bonus_spirit;
- bonus_mana;
- bonus_physical_defense;
- bonus_magic_defense;
- bonus_accuracy;
- bonus_dodge;
- bonus_crit_chance;
- bonus_crit_damage_percent;
- bonus_hp_regen_percent;
- bonus_spirit_regen_percent;
- bonus_mana_regen_percent;
- bonus_max_energy;
- bonus_energy_saving_percent;
- bonus_energy_restore_percent.

Только после сбора этих бонусов запускаются формулы характеристик.
Игроку на сайте и в боте показываются только готовые итоговые значения, без формул.
