# Чек-лист: UI/RBAC-видимость действий в админ-панели V2 (ТЗ 22)

Проверка, что frontend показывает только те действия, на которые у роли есть
backend-доступ, и что обычный отказ в правах (403) не разлогинивает админа.

Фронтенд-тест-раннера в проекте нет — это ручной чек-лист. Backend-инварианты,
на которые опирается UI, покрыты Python-тестами (view-token требует
`inventory.edit`; rollback опубликованного требует `*.publish`).

## Обработка ошибок `requestAdminJson` (§5)

- [ ] `401` — сессия очищается, сообщение «Сессия истекла. Войдите заново.».
- [ ] `403` — токен НЕ очищается, сообщение «Недостаточно прав…», админ остаётся в панели.
- [ ] `404/405` — токен НЕ очищается, сообщение «Действие недоступно для этого раздела.».
- [ ] `5xx`/сеть — токен НЕ очищается, понятное сообщение об ошибке.

## Кнопка «Импортировать библиотеку» (§1/§6)

- [ ] Видна только при `*.publish` **и** `config.supportsImport === true`.
- [ ] Конструкторы с import-route (traits/blessings/phases/races) — кнопка есть.
- [ ] Конструкторы без import-route (formulas/professions/workshops/PVP/combat/
      NPC-allies/mole/casino/housing/levels/exp/registration и пр.) — кнопки нет,
      нет запросов на несуществующий `/import` (404/405).
- [ ] Новый конструктор на `LibrarySection`: импорт показывается только если явно
      задан `supportsImport: true` и добавлен backend-route.

## Глобальный поиск (§2)

- [ ] Роль с `graph.view` — поиск отображается и работает.
- [ ] Роль без `graph.view` (moderator/support/read-only) — поиск скрыт.
- [ ] Ввод в поиск без `graph.view` не разлогинивает (поиск просто не показан).

## Карточка игрока (§3)

- [ ] Роль с `inventory.edit` — видит «Открыть редактируемый профиль».
- [ ] Роль с `players.view`, но без `inventory.edit` (support/moderator/read-only)
      — кнопки нет, edit-токен не запрашивается, нет разлогина.

## Откат версии (§4)

- [ ] Черновик/не опубликовано: rollback доступен при `*.edit`.
- [ ] Опубликовано: rollback доступен только при `*.edit` И `*.publish`.
- [ ] Editor без publish не видит rollback у опубликованной записи.
- [ ] Backend: rollback опубликованного без publish → `403`, без разлогина.

## Роли для прогона

superadmin, admin, content editor, economy editor, support, moderator, read-only.
