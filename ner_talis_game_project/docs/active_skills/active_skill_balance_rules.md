# Правила баланса активных навыков

## Главное правило открытия

Активные навыки **не открываются напрямую за уровень игрока**.

Уровень игрока может открыть только системный этап, например выбор ветви на 10 уровне у Распорядительного камня в Селдаре. После выбора ветви каждый навык открывается только по собственным требованиям:

- выбранная ветвь;
- предыдущий навык;
- уровень предыдущего навыка;
- уровень конкретного модификатора;
- вручную вложенная характеристика;
- подходящее оружие;
- наставник;
- книга;
- свиток;
- квест;
- событие;
- особое условие.

## Уровень навыка

```text
Уровень навыка = 1 + сумма уровней всех модификаторов навыка
```

Игрок не повышает сам навык напрямую. Очки навыков вкладываются только в модификаторы.

## Профильная сила навыка

```text
Профильная сила = главная характеристика × 0.65 + вторичная × 0.25 + дополнительная × 0.10
```

Для гибридных навыков профиль можно делить между частями, но обычный развиваемый навык всё равно принадлежит только одной ветви.

## Общая формула значения навыка

```text
Основа навыка = Профильная сила + Мягкий уровень × Коэффициент роли + Бонус навыка
Множитель уровня навыка = 1 + ln(Уровень навыка + 1) × 0.12
Итоговое значение = ceil(Основа × Множитель уровня навыка × Множитель модификаторов × Множитель формы)
```

Мягкий уровень может участвовать в расчётах силы навыка, но не является условием открытия навыка.

## Ресурсы

- Ветвь Духа тратит очки духа.
- Ветвь Маны тратит очки маны.
- Стартовые нейтральные действия не развивают ветви.
- Отдельного параметра «Концентрация» нет.

## Глобальные ограничения

- Снижение урона защитой не выше 70%.
- Шанс попадания от 10% до 95%.
- Шанс критического удара не выше 49%.
- Регенерация ресурсов не выше 80% от максимума.
- Обычные развиваемые навыки не должны одновременно требовать дух и ману.

## Рекомендации по модификаторам

- Сила: +4% к основному числу навыка за уровень.
- Эффект: +2 процентных пункта к шансу срабатывания или +5% к силе вторичного эффекта.
- Ресурс: -2% к расходу ресурса за уровень.
- Форма, зона, длительность, пробой, контроль и поддержка могут усиливаться линейно или порогами.


## Оружейная синхронизация

Активные навыки должны проверять только текущие типы оружия проекта: меч, кинжал, посох, топор, молот, лук, щит, арбалет.

В машинных данных используются ID: `sword`, `dagger`, `staff`, `axe`, `hammer`, `bow`, `shield`, `crossbow`.

Техническое значение `any` допустимо только для действий, не зависящих от оружия: подсумок, сбежать, часть универсальных защитных и магических действий.

Не использовать как оружие: `spear`, `focus`, `unarmed`, `any_melee`, `two_handed_sword`, `heavy_armor`, `medium_armor`. Если в будущем копьё, фокус или безоружный бой будут добавлены в проект, их нужно вводить как отдельное расширение weapon_system, а не оставлять скрытыми алиасами.


## Правило оружейных требований

`weapon_requirements` — это список, а не одиночное значение. У части навыков может быть несколько допустимых типов оружия. Проверка выполняется по правилу `any_of`: если текущее оружие игрока входит в список, навык можно использовать. `any` применяется только для универсальных действий, которые не зависят от оружия.
