# Система боя PVE — Мир Нер-Талис

**Назначение:** единый стандарт PVE-боя для Telegram/VK-бота и будущей интеграции с сайтом.

**Ключевое правило:** отдельный параметр «Концентрация» не используется. Баланс боя держится на HP, духе, мане, точности, уклонении, броне, защитах, откатах, эффектах, предметах и поведении мобов.

---

## 1. Общая логика PVE-боя

PVE-бой — пошаговый бой игрока против одного или нескольких мобов.

Бой может начаться:

- после действия «Отправиться на поиски» во внешней локации;
- при случайном событии;
- при засаде;
- при охране ресурса;
- в подземелье;
- при встрече с мини-боссом, боссом или рейдовым боссом;
- по заданию.

Энергия тратится до боя на поиск, исследование или добычу. Внутри боя энергия не расходуется.

В бою используются:

- HP;
- очки духа;
- мана;
- точность;
- уклонение;
- броня;
- физическая защита;
- магическая защита;
- откаты навыков;
- активные эффекты;
- предметы;
- особенности мобов.

## 3. Состояние игрока в бою

```yaml
player_state:
  current_hp: integer
  max_hp: integer

  current_spirit: integer
  max_spirit: integer

  current_mana: integer
  max_mana: integer

  armor: integer
  magic_armor: integer

  physical_defense: integer
  magic_defense: integer

  accuracy: integer
  dodge: integer

  temporary_shields: Shield[]
  active_effects: BattleEffect[]
  cooldowns: object
  used_items: string[]
  defense_stance: string | null
```

### Расшифровка полей игрока

| Поле | Значение |
| --- | --- |
| `armor` | Обычная броня от экипировки и эффектов. Участвует в расчёте физической защиты. |
| `magic_armor` | Магическая броня. Если отдельной магической брони нет, магическая защита может считаться от обычной брони. |
| `physical_defense` | Итоговая физическая защита после характеристик, брони, предметов, камней, сетов, баффов и эффектов. |
| `magic_defense` | Итоговая магическая защита после характеристик, магической брони, предметов, камней, сетов, баффов и эффектов. |
| `accuracy` | Итоговая точность игрока. Используется при проверке попадания. |
| `dodge` | Итоговое уклонение игрока. Используется против точности врага. |

```yaml
  role: attacker | defender | archer | mage | support | controller | summoner
  level: integer

  damage_type: physical | magic | mixed
  damage_split:
    physical: integer
    magic: integer

  current_hp: integer
  max_hp: integer

  current_spirit: integer
  max_spirit: integer

  current_mana: integer
  max_mana: integer

  armor: integer
  magic_armor: integer

  physical_defense: integer
  magic_defense: integer

  accuracy: integer
  dodge: integer

  skills: string[]
  features: string[]
```

### Расшифровка полей моба

| Поле | Значение |
| --- | --- |
| `damage_type` | Главный тип урона моба: физический, магический или смешанный. |
| `damage_split` | Используется только при `mixed`. Показывает, какая часть урона проходит через физическую, а какая через магическую защиту. |
| `armor` | Броня моба. Участвует в расчёте физической защиты. |
| `magic_armor` | Магическая броня моба. Участвует в расчёте магической защиты. |
| `physical_defense` | Итоговая физическая защита моба. |
| `magic_defense` | Итоговая магическая защита моба. |
| `accuracy` | Итоговая точность моба. |
| `dodge` | Итоговое уклонение моба. |

## 5. Тип урона мобов

В системе используется только три главных типа урона:

```yaml
damage_type:
  physical: физический урон
  magic: магический урон
  mixed: физический + магический урон
```

### Физический урон

```yaml
damage_type: physical
```

Физический урон проходит через физическую защиту цели.

Примеры мобов:

- волк;
- разбойник;
- скелет-воин;
- медведь;
- кабан;
- паук, если основной удар — укус.

### Магический урон

```yaml
damage_type: magic
```

Магический урон проходит через магическую защиту цели.

Примеры мобов:

- дух;
- элементаль;
- маг-культист;
- призрак;
- аномальная сущность.

### Смешанный урон

```yaml
damage_type: mixed
damage_split:
  physical: 70
  magic: 30
```

Смешанный урон делится на две части:

- физическая часть проходит через физическую защиту;
- магическая часть проходит через магическую защиту;
- после расчёта обе части складываются.

Примеры мобов:

- проклятый зверь Разлома;
- аномальный медведь;
- рыцарь с зачарованным оружием;
- нежить с физической атакой и тёмной силой.

## 6. Расчёт попадания

```text
Шанс попадания = accuracy / (accuracy + max(1, target.dodge))

Итоговый шанс попадания = min(0.95, max(0.10, шанс попадания))
```

Ограничения:

- минимальный шанс попадания: 10%;
- максимальный шанс попадания: 95%.

---

## 7. Расчёт снижения урона защитой

```text
Снижение урона = min(
  0.70,
  defense / (defense + 300 + soft_level * 8)
)

Итоговый урон = max(
  1,
  ceil(incoming_damage * (1 - damage_reduction))
)
```

Ограничения:

- защита не может снизить входящий урон больше чем на 70%;
- итоговый входящий урон не может быть меньше 1.

---

## 8. Расчёт физического, магического и смешанного урона

### 8.1. Физический урон

```text
final_damage = apply_defense(raw_damage, target.physical_defense, target.soft_level)
```

### 8.2. Магический урон

```text
final_damage = apply_defense(raw_damage, target.magic_defense, target.soft_level)
```

### 8.3. Смешанный урон

```text
physical_part = ceil(raw_damage * damage_split.physical / 100)
magic_part = raw_damage - physical_part

final_physical = apply_defense(physical_part, target.physical_defense, target.soft_level)
final_magic = apply_defense(magic_part, target.magic_defense, target.soft_level)

final_damage = final_physical + final_magic
```

Если `damage_type = mixed`, сумма `damage_split.physical + damage_split.magic` должна быть равна 100.

## 9. Порядок раунда

```text
1. Начало раунда.
2. Срабатывают эффекты начала раунда.
3. Игрок выбирает действие.
4. Выполняется действие игрока.
5. Проверяется смерть мобов.
6. Живые мобы выбирают действия через ИИ.
7. Выполняются действия мобов.
8. Проверяется смерть игрока.
9. Срабатывают эффекты конца раунда.
10. Уменьшаются откаты.
11. Начинается следующий раунд.
```

## 10. Кнопки боя

```text
⚔️ Обычная атака
✨ Навыки
🛡 Защита
🎒 Предмет
🏃 Отступить
📊 Состояние
```

### 10.1. Обычная атака

Базовое действие без отката.

Порядок:

1. выбрать цель;
2. проверить попадание;
3. проверить крит;
4. рассчитать урон;
5. применить защиту цели;
6. снять HP.

### 10.2. Навыки

Перед применением навыка система проверяет:

- изучен ли навык;
- подходит ли выбранная ветка развития;
- хватает ли духа или маны;
- не находится ли навык в откате;
- можно ли выбрать эту цель;
- не запрещён ли навык активным эффектом контроля.

### 10.3. Защита

Варианты:

```text
Блок
Уклонение
Парирование
Защитная стойка
```

Рекомендуемые базовые эффекты:

| Действие | Эффект |
| --- | --- |
| Блок | Снижает физический урон до конца раунда. Лучше работает со щитом. |
| Уклонение | Временно повышает `dodge` до следующей атаки врага. |
| Парирование | Шанс снизить ближний физический урон и нанести слабый ответный удар. |
| Защитная стойка | Временно повышает физическую и магическую защиту, но снижает урон игрока. |

### 10.4. Предмет

В бою можно использовать:

- зелье лечения;
- зелье духа;
- зелье маны;
- противоядие;
- бинт;
- свиток;
- метательный предмет;
- особый расходник.

Ограничение: 1 предмет за ход.

### 10.5. Отступление

Отступление можно запретить для:

- сюжетного боя;
- босса;
- ловушки;
- боя с окружением;
- состояния обездвиживания.

## 12. Эффекты и статусы

```yaml
battle_effect:
  effect_id: string
  name: string
  type: poison | bleed | burn | curse | stun | slow | weakness | healing_block | shield | mark
  duration_rounds: integer
  power: integer
  source_id: string
  can_stack: boolean
  stack_count: integer
  max_stacks: integer
  remove_condition: string | null
```

Базовые статусы:

| Статус | Эффект |
| --- | --- |
| Яд | Периодический урон каждый раунд. |
| Кровотечение | Физический периодический урон. |
| Горение | Магический периодический урон, может снижать защиту. |
| Оглушение | Цель пропускает ход. |
| Замедление | Снижает уклонение и шанс побега. |
| Слабость | Снижает урон. |
| Проклятие | Снижает восстановление, защиту или накладывает особое условие. |
| Блок лечения | Запрещает восстановление HP. |
| Щит | Поглощает часть урона. |
| Метка | Повышает шанс попадания или крита по цели. |

## 14. Победа

После победы:

```text
1. Бой завершается.
2. Игрок получает опыт.
3. Считается добыча с каждого врага.
4. Применяется групповой штраф добычи, если врагов было несколько.
5. Предметы добавляются в инвентарь.
6. Монеты добавляются на баланс.
7. Состояние боя очищается.
8. Игрок возвращается в локацию.
```

---

## 15. Поражение

Мягкий вариант поражения:

```text
1. Бой завершается.
2. Игрок переносится в безопасную точку.
3. HP становится 1.
4. Добыча текущего поиска теряется.
5. Игрок получает временный эффект «Травма».
```

Пример травмы:

```yaml
injury_effect:
  duration_minutes: 30
  max_hp_percent: -10
  accuracy_percent: -10
  dodge_percent: -10
```

```text
9. Победу.
10. Поражение.

```

После этого добавить:

```text
1. Навыки игрока.
2. Навыки мобов.
3. Эффекты и статусы.
4. Предметы в бою.
5. Побег.
6. ИИ мобов по ролям.
7. Фазы боссов.
8. Подземелья и волны врагов.
```

## 17. Итоговое правило совместимости

Система PVE-боя должна использовать единые поля персонажа и мобов:

```text
HP
Дух
Мана
Броня
Магическая броня
Физическая защита
Магическая защита
Точность
Уклонение
Тип урона моба: physical / magic / mixed
```

Отдельный параметр «Концентрация» не используется.
