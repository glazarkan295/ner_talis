# ТЗ для Claude Code: реферальные ссылки в профиле игрока для Telegram и VK

## 1. Задача

В профиле игрока Нер-Талис нужно доработать раздел реферальных ссылок.

Сейчас в профиле есть или планируется ссылка только на Telegram-бота. Нужно сделать полноценную систему реферальных ссылок для двух платформ:

```text id="ouvm6v"
Telegram-бот
VK-бот
```

Реферальные ссылки должны находиться в отдельной вкладке внутри раздела:

```text id="vf21ld"
Профиль игрока → Сервисы → Реферальные ссылки
```

Нужно не просто показать ссылки, а обеспечить их работоспособность:

```text id="aksxjx"
генерация ссылки
копирование ссылки
переход по ссылке
обработка параметра в Telegram
обработка параметра в VK
привязка приглашённого игрока к пригласившему
защита от саморефералов и дублей
отображение статистики приглашений
```

---

## 2. Где должна находиться вкладка

В профиле игрока в разделе:

```text id="6r8yts"
Сервисы
```

добавить отдельную вкладку:

```text id="8cictp"
Реферальные ссылки
```

Важно:

* не размещать реферальные ссылки внутри «Передачи»;
* не смешивать рефералку с доставкой предметов;
* не делать отдельную главную вкладку профиля, чтобы не захламлять интерфейс;
* вкладка должна быть внутри «Сервисов».

Структура:

```text id="4fsmu6"
Профиль
 └─ Сервисы
     ├─ Передача
     ├─ Торговый павильон, если доступен
     └─ Реферальные ссылки
```

Блок:


```text id="9nyg39"

```

Важно: код должен поддерживать оба варианта.

```python id="k6x3vl"

```

Требования:

```text id="bcsu81"

код не меняется при каждом открытии профиля

код не содержит личных данных игрока
код безопасен для URL
код короткий и читаемый
```

Пример:

```text id="vhkbc5"

```

или:

```text id="mbqgii"

```

```text id="yewtii"

vk_id
email
токен профиля

внутренний секрет
```

## 6. ENV-настройки

Добавить или проверить переменные окружения:

```env id="5a4ob5"
TG_BOT_USERNAME=NerTalisBot

VK_GROUP_ID=123456789
VK_COMMUNITY_SCREEN_NAME=ner_talis_game
VK_REF_LINK_TEMPLATE=https://vk.me/{VK_COMMUNITY_SCREEN_NAME}?ref=ref_{REF_CODE}
VK_REF_FALLBACK_LINK_TEMPLATE=https://vk.com/im?sel=-{VK_GROUP_ID}&msg=/start%20ref_{REF_CODE}

REFERRALS_ENABLED=true
REFERRAL_CODE_PREFIX=ref_
```

Требования:

* `TG_BOT_USERNAME` нужен для генерации Telegram-ссылки;
* `VK_GROUP_ID` нужен для fallback-ссылки VK;
* `VK_COMMUNITY_SCREEN_NAME` нужен для короткой VK-ссылки;
* если чего-то не хватает, показывать в профиле понятный статус «ссылка временно недоступна»;
* не падать с ошибкой 500.

## 7. Обработка Telegram-рефералки

В Telegram-боте обработать команду:

```text id="axvk0d"
/start ref_{REF_CODE}
```

Пример:

```text id="hed25c"
/start ref_AB12CD34
```

Логика:

1. получить payload из `/start`;
2. проверить, что payload начинается с `ref_`;
3. извлечь `REF_CODE`;
4. найти пригласившего игрока по `referral_code`;
5. проверить, что это не сам игрок;
6. если пользователь новый — сохранить pending referral;
7. после завершения регистрации засчитать приглашение;
8. если пользователь уже зарегистрирован — не менять старого реферера;
9. записать событие в историю.

---

## 8. Обработка VK-рефералки

В VK-боте нужно поддержать несколько вариантов, потому что ссылка может прийти в разных форматах.

### 8.1. Через ref-параметр

Если событие VK содержит поле:

```text id="b9psdv"
ref
```

или аналогичный payload/context, обработать:

```text id="5dhf9o"
ref_AB12CD34
```

### 8.2. Через текст сообщения

Поддержать команды:

```text id="v8muyx"
/start ref_AB12CD34
start ref_AB12CD34
ref_AB12CD34
начать ref_AB12CD34
```

Логика такая же, как в Telegram:

```text id="o86of1"
извлечь REF_CODE
найти пригласившего
создать pending referral
после регистрации засчитать приглашение
```

## 9. Pending referral

Если пользователь пришёл по ссылке, но ещё не завершил регистрацию, сохранить временную привязку:

```python id="jbb5zj"
pending_referral = {
    "platform": "telegram | vk",
    "external_user_id": "...",
    "referral_code": "AB12CD34",
    "referrer_game_id": "NT-...",
    "created_at_ts": 1782628810,
    "expires_at_ts": 1783233610
}
```

Срок жизни pending referral:

```text id="adshlc"
7 дней
```

Если игрок зарегистрировался позже срока — приглашение не засчитывать.

---

## 10. Засчитывание приглашения

При завершении регистрации нового игрока:

1. проверить pending referral;
2. найти пригласившего;
3. проверить антиабуз-правила;
4. записать связь:

```python id="m5hxqz"
new_player["referred_by"] = referrer_game_id
```

5. добавить нового игрока в список приглашённых у реферера:

```python id="inmeyy"
referrer["referrals"].append({
    "player_game_id": new_player_game_id,
    "platform": "telegram | vk",
    "created_at_ts": now_ts,
    "status": "registered"
})
```

6. записать историю;
7. при наличии механики наград — начислить награду по событию регистрации.

---

## 11. Защита от злоупотреблений

Нужно запретить:

```text id="t9qxgi"
самореферал
повторную привязку реферера
смену реферера после регистрации
засчитывание приглашения уже существующего игрока
дубли одного и того же приглашённого
засчитывание по несуществующему referral_code
```

Правила:

```text id="2te4l6"
один игрок может иметь только одного referred_by
один referral_code принадлежит только одному игроку
если игрок уже зарегистрирован, новый переход по ссылке только логируется, но не меняет referred_by
если referrer_game_id == new_player_game_id — не засчитывать
```

## 12. Кроссплатформенность

Реферальная система должна работать с единым NT-ID.

Сценарии:

### 12.1. Игрок приглашает через Telegram, друг регистрируется в Telegram

Засчитывается.

### 12.2. Игрок приглашает через VK, друг регистрируется в VK

Засчитывается.

### 12.3. Игрок приглашает через Telegram, друг позже привязывает VK

Рефералка остаётся той же, дубль не создаётся.

### 12.4. Игрок приглашает через VK, друг позже привязывает Telegram

Рефералка остаётся той же, дубль не создаётся.

### 12.5. Один и тот же человек переходит по Telegram и VK ссылке

Должен быть только один `referred_by`.


## 20. Интеграция с VK-ботом

1. проверить `event.object.message.ref`, если поле доступно;

3. извлечь `ref_CODE`;

Пример поддерживаемых сообщений:

## 24. Ручная проверка

Проверить вручную.

### 24.1. Telegram

1. открыть профиль игрока;
2. перейти в `Сервисы → Реферальные ссылки`;
3. скопировать Telegram-ссылку;
4. открыть ссылку другим Telegram-аккаунтом;
5. нажать Start;
6. завершить регистрацию;
7. проверить, что приглашение засчитано.

### 24.2. VK

1. открыть профиль игрока;
2. перейти в `Сервисы → Реферальные ссылки`;
3. скопировать VK-ссылку;
4. открыть ссылку другим VK-аккаунтом;
5. написать/нажать старт в сообщениях сообщества;
6. завершить регистрацию;
7. проверить, что приглашение засчитано.

### 24.3. Проверка дублей

1. тем же игроком снова открыть ссылку;
2. убедиться, что повторное приглашение не создаётся.

## 25. Критерии готовности

Готово, если:

1. В профиле есть вкладка `Сервисы → Реферальные ссылки`.
2. Во вкладке есть Telegram-реферальная ссылка.
3. Во вкладке есть VK-реферальная ссылка.
4. Обе ссылки используют один стабильный `referral_code` игрока.
5. Telegram-ссылка открывает Telegram-бота и передаёт `ref_CODE`.
6. VK-ссылка открывает сообщения VK-бота и позволяет передать `ref_CODE`.
7. Telegram-бот обрабатывает `/start ref_CODE`.
8. VK-бот обрабатывает `ref_CODE` через payload или текстовую команду.
9. После регистрации новый игрок получает `referred_by`.
10. У пригласившего появляется запись о приглашённом игроке.
11. Самореферал невозможен.
12. Повторное засчитывание невозможно.
13. Старые игроки без referral_code автоматически получают код.
14. В админ-панели можно посмотреть referral_code, referred_by и список приглашённых.
15. Тесты покрывают Telegram, VK, генерацию ссылок, регистрацию, самореферал и дубли.

## 26. Краткая формулировка задачи

Доработать реферальную систему профиля игрока: создать отдельную вкладку `Сервисы → Реферальные ссылки`, показывать две ссылки — на Telegram-бота и VK-бота, генерировать единый безопасный referral_code, обрабатывать переходы по ссылкам в Telegram и VK, засчитывать приглашения после регистрации и защищаться от саморефералов, дублей и нерабочих ссылок.
