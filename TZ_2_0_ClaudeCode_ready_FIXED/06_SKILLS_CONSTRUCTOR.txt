# 2. Назначение конструктора навыков

Конструктор навыков нужен для полного управления всеми навыками, приёмами, заклинаниями, пассивными умениями, боевыми действиями, предметными навыками, навыками мобов, навыками за достижения и особыми навыками.

Через конструктор админ должен уметь создавать, редактировать, публиковать и связывать навыки с:

* игроками;
* ветками Маны;
* ветками Духа;
* путями развития;
* предметами;
* оружием;
* магическими книгами;
* экипировкой;
* мобами;
* NPC;
* достижениями;
* квестами;
* эффектами;
* формулами;
* боевой системой;
* событиями;
* локациями;
* расами;
* репутацией;
* скрытой репутацией.

Главное требование: опубликованный навык должен реально работать в игре и использоваться через ID в других конструкторах.

# 4. Главная структура конструктора

## 4.1. Разделение навыков

В конструкторе обязательно сделать разделение на группы:

1. **Навыки ветки Маны**
   Магические навыки, заклинания, чародейские действия, эффекты, завязанные на ману.

2. **Навыки ветки Духа**
   Физические приёмы, стойки, боевые техники, усиления тела, действия за ресурс Дух.

3. **Навыки по путям**
   Внутренние направления развития внутри веток Маны и Духа.

4. **Навыки предметов**
   Навыки, которые появляются от оружия, брони, артефактов, магических книг, свитков, экипировки или других предметов.

5. **Навыки мобов**
   Навыки, которые могут использовать только мобы, боссы, уникальные противники и NPC-противники.

6. **Навыки за достижения**
   Навыки, которые открываются после получения достижения.

7. **Особые навыки**
   Уникальные, сюжетные, системные, событийные, расовые, квестовые или админские навыки.

# 6. ID навыка

Поле:

* ID навыка.

Требования:

* ID обязателен;
* ID должен быть уникальным;
* ID используется в бою, предметах, мобах, достижениях, событиях, NPC и профиле игрока;
* опубликованный ID нельзя менять без проверки связей;
* если навык уже есть у игроков, предметов, мобов или достижений, изменение ID должно быть запрещено либо должно обновлять все связи автоматически;
* в карточке навыка должна быть кнопка «Где используется».

* навыки должны проверять ресурс;

* навыки должны проверять боеприпасы;

* оглушение запрещает основные и дополнительные действия;
* травма руки может блокировать оружейный слот;
* оковы времени увеличивают откаты навыков;
* чистый урон игнорирует защиту;

# 34. Публикация навыка

## 34.1. Проверка перед публикацией

* заполнен ID;
* ID уникален;
* заполнено название;
* выбрана категория;
* выбран тип навыка;
* выбрана ветка, если навык относится к Мане или Духу;
* выбран путь, если навык относится к пути;
* указана стоимость применения или явно включено бесплатное применение;
* указана цель;
* указано действие навыка;
* все эффекты существуют;
* все формулы существуют;
* все предметы существуют;
* все мобы существуют;
* все достижения существуют;
* все кнопки существуют;
* откат не отрицательный;
* стоимость не отрицательная;
* шанс от 0 до 100;
* длительность не отрицательная;
* если требуется оружие, тип оружия указан;
* если требуется магическая книга, указан тип требования.

---

## 34.2. Ошибки публикации

Публикацию запрещать, если:

* нет ID;
* нет названия;
* нет категории;
* нет типа;
* навык Маны не имеет настроек Маны;
* навык Духа не имеет настроек Духа;
* навык ссылается на несуществующий эффект;
* навык ссылается на несуществующую формулу;
* навык ссылается на несуществующий предмет;
* навык ссылается на несуществующего моба;
* навык ссылается на несуществующее достижение;
* шанс меньше 0 или больше 100;
* стоимость ресурса отрицательная;
* откат отрицательный;
* длительность отрицательная;
* навык моба доступен игрокам без разрешения;
* предметный навык не имеет предмета-источника;
* навык достижения не имеет достижения-источника.

---

## 34.3. Предупреждения

Показывать предупреждения:

* навык опубликован, но нигде не используется;
* навык не имеет иконки;
* навык не имеет полного описания;
* навык имеет стоимость, но нет текста нехватки ресурса;
* навык имеет откат, но нет текста отката;
* навык требует оружие, но не указан текст отсутствия оружия;
* навык требует магическую книгу, но нет текста ошибки;
* навык имеет эффект, но не указана длительность;
* навык моба не привязан ни к одному мобу;
* навык достижения не привязан к достижению;
* особый навык скрыт, но не имеет условия открытия;
* навык использует формулу в черновике.

# 35. Критерии приёмки конструктора навыков

Задача считается выполненной, если:

1. В админ-панели появился раздел **Библиотека → Бой и механики → Навыки**.
2. Есть подразделы навыков Маны, Духа, путей, предметных навыков, навыков мобов, навыков за достижения и особых навыков.
3. Старые навыки проекта импортируются в конструктор.
4. Старые ID навыков сохраняются.
5. Новый навык можно создать вручную.
6. Навык можно отнести к ветке Маны.
7. Навык можно отнести к ветке Духа.
8. Навык можно привязать к пути развития.
9. Навык можно сделать активным.
10. Навык можно сделать пассивным.
11. Навык можно сделать предметным.
12. Навык можно привязать к предмету.
13. Навык можно привязать к мобу.
14. Навык можно привязать к достижению.
15. Навык можно сделать особым.
16. Навык можно связать с эффектом.
17. Навык можно связать с формулой.
18. Навык можно связать с кнопкой.
19. Навык проверяет стоимость применения.
20. Навык проверяет откат.
21. Навык проверяет требования оружия.
22. Навык проверяет требования магической книги.
23. Навык проверяет боеприпасы.
24. Навык проверяет состояние игрока.
25. Навык может наносить урон.
26. Навык может лечить.
27. Навык может восстанавливать ману, дух или энергию.
28. Навык может накладывать эффекты.

41. Все ошибки и предупреждения отображаются на русском языке.
42. Опубликованный навык реально работает в Telegram/VK-боте.
