# README для Claude Code — порядок чтения ТЗ 2.0

## Статус пакета

Пакет исправлен и подготовлен для передачи в Claude Code. Критичные проблемы из аудита закрыты:

1. `Предметы` добавлены в единый авторитетный список как конструктор №1.
2. Качество `Небесный` добавлено между `Мифический` и `Божественный` в списки качеств предметов и результатов ремесла.
3. Очередь и приоритет сообщений вынесены в отдельный системный модуль `18_MESSAGE_QUEUE_PRIORITY.txt`.
4. Специальные файлы `14`–`17` помечены как уточнения к основным конструкторам, а не как отдельные конкурирующие версии.

## Как читать пакет

1. Сначала прочитать `00_GENERAL_REQUIREMENTS.txt` — общие требования ко всей админ-панели, сайту, профилю, очередям сообщений и лимитам.
2. Затем прочитать `00_README_FOR_CLAUDE_CODE.md` и `AUDIT_REPORT.md` — они фиксируют порядок и статус исправлений.
3. Затем читать файлы по порядку номеров `01`–`13` — основные конструкторы и крупные подсистемы.
4. Затем прочитать `18_MESSAGE_QUEUE_PRIORITY.txt` — обязательный сквозной модуль сообщений для всех конструкторов.
5. Файлы `14`–`17` — точечные/детальные ТЗ, которые уточняют отдельные подсистемы: клавиатуру Telegram, передачу, реферальные ссылки и штрафы.
6. `01_EFFECTS_SOURCE_LIST.txt` — исходный список эффектов. Его нужно использовать как источник стартовых пресетов для конструктора эффектов из `04_EFFECTS_STATES_FORMULAS.txt`.

## Обязательные сквозные требования

Для всех основных конструкторов должны быть:

- уникальный ID сущности;
- русские названия типов, статусов, категорий, условий и действий;
- статусы: черновик, на проверке, опубликовано, отключено, архив, ошибка проверки;
- поиск и фильтры;
- предпросмотр;
- история изменений;
- проверка перед публикацией;
- кнопка «Где используется»;
- защита от удаления используемых сущностей;
- импорт старых данных без потери ID и связей;
- работа опубликованных сущностей в Telegram/VK-боте без ручного изменения кода;
- подключение к системной очереди сообщений, если конструктор отправляет сообщения игроку, админу или в общий чат.

## Особые уточнения, которые нельзя потерять

- Конструктор предметов является первым и обязательным конструктором.
- Качество предметов должно включать уровень `Небесный` между `Мифический` и `Божественный`.
- Все сообщения бота должны поддерживать очередь, таймеры и приоритеты через модуль `18_MESSAGE_QUEUE_PRIORITY.txt`.
- Сайт и профиль должны работать только через безопасную активную HTTPS-ссылку.
- Профиль сохраняет стиль старого профиля, не использует модель персонажа, не показывает формулы игроку.
- Подпольное казино работает только с внутриигровыми валютами/предметами и не имеет вывода реальных денег.
- PVE-бой должен поддерживать побег мобов по условиям.
- PVP-посмертные проклятья с достижением «Проклятье? Какое проклятье?» относятся только к PVP-смерти от игрока, не к мобам/предметам/ловушкам/событиям.
