# Часть 1. Конструктор профиля игрока

## 2. Назначение конструктора профиля

Конструктор профиля нужен для настройки внешнего вида, вкладок, блоков, кнопок, отображения данных и доступных действий в профиле игрока.

Профиль должен быть связан с Telegram/VK-ботом и открываться по активной ссылке сайта через одноразовый токен.

Конструктор профиля должен позволять администратору управлять:

* вкладками профиля;
* порядком вкладок;
* отображением характеристик;
* отображением экипировки;
* отображением инвентаря;
* отображением навыков;
* отображением эффектов;
* отображением штрафов;
* отображением достижений;
* сервисами игрока;
* передачей предметов;
* будущим торговым павильоном;
* кнопками;
* текстами;
* стилем;
* видимостью блоков;
* правами действий;
* предпросмотром профиля.

Главное требование: профиль должен сохранять стилистику и цветовую атмосферу старого профиля, но стать удобнее, понятнее и гибче.

# 7. Стиль и оформление профиля

## 7.1. Общие требования к стилю

Профиль должен:

* сохранять стилистику старого профиля;
* сохранять цветовую палитру старого профиля;
* не менять визуальную атмосферу проекта;
* быть удобным на телефоне;
* не захламлять экран;
* не показывать формулы игроку;
* показывать данные простым языком;
* иметь аккуратные вкладки;
* иметь читаемые кнопки;
* иметь нормальный размер изображений;
* не растягивать превью предметов на весь экран.

---

## 7.2. Поля оформления

Поля:

* основной цвет;
* вторичный цвет;
* цвет фона;
* цвет карточек;
* цвет текста;
* цвет положительных значений;
* цвет отрицательных значений;
* цвет предупреждений;
* цвет опасной зоны;
* цвет кнопок;
* стиль рамок;
* скругление блоков;
* размер иконок;
* размер изображений предметов;
* режим компактного отображения;
* режим подробного отображения.

---

## 7.3. Ограничение по модели персонажа

В профиле не должна использоваться визуальная модель персонажа.

Не добавлять:

* модель персонажа по расе;
* модель персонажа по полу;
* обязательный визуальный аватар тела;
* блок с 3D/2D-моделью персонажа.

Профиль должен показывать игровые данные без модели персонажа.

## 8.2. Правила вкладок

Правила:

* вкладку «Обзор» не добавлять;
* вкладку «Сервисы» показывать только если игроку доступен хотя бы один сервис;
* «Передача» должна находиться внутри «Сервисов»;
* «Торговый павильон» должен находиться внутри «Сервисов»;
* «Гильдии» зарезервировать на будущее, но пока скрыть;
* «Коллекции» не добавлять;
* книги, записки и картины должны быть частью будущей библиотеки дома, а не отдельной вкладкой профиля.

# 11. Экипировка

## 11.1. Слоты экипировки

Профиль должен поддерживать 12 слотов экипировки.

Поля слота:

* ID слота.
* Название слота.
* Иконка.
* Типы предметов.
* Занят / свободен.
* Заблокирован.
* Причина блокировки.
* Предмет в слоте.
* Кнопки действий.

---

## 11.2. Двуручное оружие

Правило:

* двуручное оружие блокирует второй слот руки;
* если второй слот занят, надеть двуручное оружие нельзя;
* если надето двуручное оружие, второй слот должен показываться как заблокированный;
* текст блокировки должен редактироваться.

---

## 11.3. Травмы рук

Профиль должен учитывать эффекты:

* травма правой руки;
* травма левой руки.

Правила:

* травма правой руки может блокировать оружие 1;
* травма левой руки может блокировать оружие 2;
* в профиле показывать причину блокировки;
* кнопки использования заблокированного слота должны быть недоступны.

---

* скрыть «Использовать»;
* скрыть «Выбросить»;
* скрыть опасные игровые действия;
* запретить изменение характеристик из профиля;
* запретить изменение навыков из профиля;
* разрешить только просмотр данных;
* отдельные админские действия делать только через админ-панель с правами.

# 17. Рейтинги в профиле

В профиле показывать только места самого игрока.

Примеры:

* место по достижениям;
* место по PVE;

Полные рейтинги не показывать в профиле.
Полные рейтинги должны быть на отдельной странице сайта, если будут включены.

# 18. Сервисы профиля

Вкладка «Сервисы» показывается только если игроку доступна хотя бы одна функция.

Сервисы:

* Передача.
* Торговый павильон.
* Другие будущие сервисы.

# 19. Передача предметов

Передача должна находиться внутри вкладки «Сервисы».

## 19.1. Функции передачи

Поля:

* поиск игрока по нику;
* поиск игрока по NT-ID;
* поиск предметов;
* выбор количества;
* список выбранных предметов;
* удаление предмета из списка;
* короткое сообщение до 30 символов;
* предупреждение о доставке 10–15 минут;
* отображение итоговой стоимости;
* подтверждение отправки;
* сообщение получателю.

---

## 19.2. Стоимость передачи

Игроку показывать только итоговую стоимость.

Формулы стоимости не показывать.

Примеры отображения:

* 10 мед.
* 1 сер. 339 мед.
* 1 зол. 115 сер. 869 мед.

---

## 19.3. Сообщение получателю

Получатель должен получить сообщение в чат:

**Вы получили посылку от игрока: ...**

Если предмет выдан админом или через рассылку:

**Вы получили в дар от высших сил: ...**

# 20. Торговый павильон

Торговый павильон — будущий сервис профиля.

Правила:

* не делать отдельной вкладкой верхнего уровня;
* показывать внутри «Сервисов»;
* показывать только если игроку доступна функция павильона;
* доступ может быть по покупке, аренде, квесту, достижению или админскому флагу.

Поля:

* павильон доступен;
* павильон куплен;
* павильон арендован;
* срок аренды;
* лимит товаров;
* комиссия;
* список товаров;
* история продаж.

# 23. Безопасность профиля

Требования:

* профиль открывается по одноразовому токену из бота;
* при выдаче нового токена старый отключается;
* после активации токена использовать Authorization: Bearer;
* токен нужно убрать из URL после активации;
* не хранить токен в небезопасном виде;
* публичный identifier не должен отдавать полный JSON профиля;
* нужна защита от перебора;
* нужен rate-limit;
* нужен HTTPS;
* нужны безопасные заголовки;
* админский read-only режим не должен позволять игровые действия.

## 31.3. Правила активной ссылки

Правила:

* ссылка должна начинаться с `https://`;

* бот должен использовать только опубликованную активную ссылку;
* нельзя использовать старую ссылку после смены активной ссылки;
* при смене активной ссылки новые кнопки должны использовать новый URL;
* старые одноразовые токены профиля должны работать только по правилам безопасности;
* если сайт недоступен, админка должна показывать предупреждение;
* если ссылка неактивна, кнопка «Открыть профиль» должна показывать понятную ошибку.

* домен разрешён;
* сайт отвечает;
* профиль доступен;
* страница ошибки работает;
